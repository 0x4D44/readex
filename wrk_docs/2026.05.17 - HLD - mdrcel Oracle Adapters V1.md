## 1. The whole design in one paragraph

Two tiny programs — `run.py` (Trafilatura) and `run.mjs` (Readability-JS) — each take an HTML file path, run one well-validated extractor, and print **exactly one JSON object** describing what that extractor found. They share no code; they share one thing only: a **frozen JSON contract** and a **byte-exact word-count rule**, both committed as data, not code. A single Rust integration test (`cargo test`) feeds both adapters a handful of synthetic fixtures and asserts schema, `ok`/exit semantics, and run-to-run determinism. That test *is* the proof that "the contract is frozen." Everything else in this document exists to make those three artifacts (two adapters + one contract) correct and reproducible. If you cannot hold that in your head, the design is wrong — so the rest of this is detail, not structure.

```
                      benchmark/oracles/contract.schema.json   (the seam — frozen data)
                                        |
        run.py  <snapshot.html> ──┐     |     ┌── run.mjs <snapshot.html>
        (Python venv, pinned)     ├──── contract ────┤   (Node, pinned)
                                  │   one JSON obj    │
                                  │   on stdout       │
                                  v                   v
                 tests/oracle_contract.rs  ──  "oracle of the oracles"
                 (cargo test: schema + ok-semantics + determinism)
                                  |
                                  v
                 (downstream, OUT OF SCOPE) Rust runner reuses this exact seam
```

## 2. What this delivers (and what it deliberately does not)

**In scope:** the contract schema artifact; `benchmark/oracles/trafilatura/`; `benchmark/oracles/readability-js/`; the canonical word-count spec; the Rust conformance harness with a small synthetic fixture set; the pinning/reproducibility mechanics; per-adapter READMEs.

**Out of scope (owned elsewhere, do not build here):** the Rust *extraction crate*; the Rust *runner* and *compare/report* layer; similarity metrics; the real 50+ URL corpus and the corpus snapshot tool; the gold set; any extraction algorithm work; JS rendering, network fetching, encoding detection (caller's job per grandparent brief).

**Non-goals by user direction (2026-05-17):** no migration/rollback machinery (dev sandbox); no environment-variable or feature-flag switches anywhere in the adapters or harness — behaviour is fixed by committed files, not toggled at runtime.

## 3. The frozen contract (this is the seam — freeze before parallel work)

The contract is committed as a **machine-checkable JSON Schema** at `benchmark/oracles/contract.schema.json`. This is the single shared artifact. It is *data, not code*, so it does not violate the brief's "no shared code between the two adapters" rule — it **is** the thing they must both satisfy, and making it a file (not prose) turns "the contract is frozen" into a checkable claim.

### 3.1 Invocation

```
<entrypoint> <path-to-snapshot.html> [--base-url <URL>]
```

* The single positional arg is a filesystem path. The adapter reads it as **raw bytes** and hands those bytes to the underlying library **unmodified** — it performs **no** encoding detection, charset transcoding, or normalization of its own. Snapshots are expected to be UTF-8 (the downstream corpus tool's responsibility). This is the grandparent brief's scope boundary, honoured literally.
* No network access, ever. No subresource fetching, no DNS, no clock-dependent or locale-dependent values in any emitted field.
* `--base-url` is optional. When present it is passed to the underlying tool's base-URL parameter and used only for canonical/relative-URL resolution. When absent, `canonical_url` may be `null` or a relative string exactly as the tool reports it.

### 3.2 Output — exactly one JSON object on stdout

stdout MUST contain **one** UTF-8 JSON object and nothing else. All logging, warnings, parser noise, and stack traces go to **stderr**. Field order is fixed (below) so committed baselines diff cleanly.

```json
{
  "oracle": "trafilatura | readability-js",
  "oracle_version": "<exact upstream version string>",
  "title": "string | null",
  "text": "string",
  "html": "string | null",
  "word_count": 0,
  "canonical_url": "string | null",
  "language": "string | null",
  "ok": true,
  "error": "string | null"
}
```

`contract.schema.json` pins: required keys (all ten, always present), types, the `oracle` enum, `ok` boolean, and the rule `ok == (error == null)`. The conformance harness validates every adapter output against this schema.

### 3.3 The `ok` / `error` / exit-code split (this encodes Bug E2's lesson)

Three states, never conflated:

| State | stdout | `ok` | `error` | exit |
|---|---|---|---|---|
| Extractor ran, found content | full JSON | `true` | `null` | `0` |
| Extractor ran, **found little/nothing** | full JSON, `text:""` (or short) | `true` | `null` | `0` |
| **Adapter itself failed** (unreadable file, tool exception, broken env it can catch) | full JSON, fields null/empty | `false` | message | non-zero |

Design rules:

1. **"Found little content" is a valid result, not a failure.** Readability `.parse()` returning `null`, or Trafilatura returning no content, maps to `ok:true` + `text:""` + exit `0`. This is the exact distinction Bug E2 collapsed; do not re-collapse it.
2. **The JSON envelope is emitted on stdout even on the failure path** — belt-and-braces. The runner can therefore always parse one uniform shape from stdout, *and* independently distinguish a crash via the non-zero exit. Dual signalling is deliberate.
3. **Uncatchable failures are out of the adapter's hands and honestly so.** If the interpreter/runtime is absent or the script can't start, there is no JSON and a non-zero exit. The conformance harness treats "no parseable stdout" as a distinct catastrophic state, separate from `ok:false`. We do not pretend the adapter can catch what kills it before it runs.

### 3.4 Determinism

Same input bytes → **byte-identical JSON** across runs and machines. Guaranteed by: no network, no clock/locale/random in emitted fields, fully pinned dependency closures (§7), and explicit (not ambient) tool configuration (§5, §6). Verified same-machine by the conformance harness; cross-machine assured by pinning.

## 4. The shared word-count rule (single source of truth)

`word_count` MUST be **recomputed by the adapter over its own `text` field** using one byte-exact rule — never taken from the tool's internal counter, or coverage numbers across oracles (and later the crate) are not comparable. The brief defers the canonical spec to the methodology note, which does not exist yet. To unblock parallel adapter work **now**, the rule lives at `benchmark/oracles/WORDCOUNT.md` as the authoritative interim source; when the methodology note is written it absorbs this verbatim and `WORDCOUNT.md` becomes a pointer to it.

### 4.1 Why this needs an explicit code-point set, not `\s`

"Split on Unicode whitespace" is a trap across languages — bare regex `\s` is **not** the Unicode `White_Space` property and the engines disagree with each other:

* ECMAScript `\s` **includes** U+FEFF (ZWNBSP) and **excludes** U+0085 (NEL).
* Python `re` `\s` and `str.split()` use yet another set (and differ from each other).
* Unicode `White_Space=Yes` is a fixed set of 25 code points and includes **neither** the JS quirk nor matches the Python quirk.

Two independent implementations using `\s` would silently produce different counts. So the rule is defined by an **explicit enumerated set**, embedded identically in both adapters (necessary duplication — there is no shared code), with `WORDCOUNT.md` as the reference both must mirror.

### 4.2 The rule

Separator set `S` = exactly these code points (Unicode `White_Space=Yes`):

```
U+0009 U+000A U+000B U+000C U+000D   (TAB LF VT FF CR)
U+0020                               (SPACE)
U+0085                               (NEL)
U+00A0                               (NBSP)
U+1680                               (OGHAM SPACE MARK)
U+2000 .. U+200A                     (EN QUAD .. HAIR SPACE)
U+2028 U+2029                        (LINE / PARAGRAPH SEPARATOR)
U+202F                               (NARROW NO-BREAK SPACE)
U+205F                               (MEDIUM MATHEMATICAL SPACE)
U+3000                               (IDEOGRAPHIC SPACE)
```

`word_count` = the number of maximal non-empty runs of characters **not** in `S`. Equivalently: count the matches of the character class `[^S]+` over `text`. No Unicode normalization, no case folding, no trimming. Empty or all-separator `text` → `0`.

Regex class to embed (both adapters): `[	-    -     　]`. Counting `[^…]+` matches (not `split`) avoids the leading/trailing empty-token edge entirely and is unambiguous in both Python's `re` and JS's `RegExp` because every member is given by explicit code point — no reliance on any engine's Unicode-property tables.

### 4.3 Proving parity

Parity is proven end-to-end, not by inspection: one conformance fixture ("whitespace stress") has a body engineered so the *extracted* `text` contains NBSP, NEL, an em-space, an ideographic space, and a stray U+FEFF (which must **not** split). The harness asserts both adapters return the **same** `word_count` for it **and** that it equals the value `WORDCOUNT.md` specifies.

## 5. Trafilatura adapter — `benchmark/oracles/trafilatura/run.py`

**Responsibility:** read bytes → run Trafilatura with an explicit pinned configuration → map to the contract → print one JSON object.

**Design constraints (not a verified call sequence — the exact API calls are pinned and verified against the pinned version at build time by the conformance harness; do not hand-tune from memory):**

* **Use Trafilatura's standard extraction with metadata** so `title`, `text`, and the tool's reported `canonical`/`language` come from one call. `text` is the tool's plain-text extraction, used verbatim as the contract's `text` (the primary comparison surface).
* **Explicit configuration, never ambient.** The adapter MUST pass an explicit Trafilatura configuration committed alongside it and MUST NOT inherit any user-level or working-directory config. Network/“fetch” paths are unused (we pass bytes). The configuration is part of the frozen artifact set.
* **Pin the whole algorithm, not just the package.** Trafilatura's benchmarked state-of-the-art result (Barbaresi 2021) is the *cascade including its fallback extractors*. Whether those fallbacks are present changes output. Therefore the **entire transitive dependency closure is pinned** (§7), so the algorithm represented is fixed and matches the benchmarked configuration. The chosen fallback behaviour is recorded in the adapter README.
* **Field mapping:**
  * `oracle` = `"trafilatura"`; `oracle_version` = the installed Trafilatura version string, read at runtime from the package (so a version bump shows up as a baseline diff).
  * `title`, `text` = from the tool's extraction, as-is.
  * `html` = Trafilatura's content HTML **only if a single extraction call yields it cheaply**; otherwise `null`. `html` is explicitly not a comparison surface and not v1-gating (brief), so asymmetry with the JS adapter here is acceptable and documented.
  * `word_count` = recomputed from `text` by §4 (NOT Trafilatura's count).
  * `canonical_url` = the tool's reported canonical/source URL, resolved against `--base-url` when that resolves a relative value; else `null`.
  * `language` = best-effort: the tool's reported language **only if available without enabling a statistical language-detection dependency**; else `null`. We deliberately do **not** add a langid dependency — language is explicitly low-priority/null-acceptable, and avoiding it keeps the closure small, fast, and deterministic.
* **Error handling:** unreadable path / Trafilatura raising / import failure the script can catch → emit the full JSON envelope with `ok:false`, `error` set, other fields null/empty, and a non-zero exit. A clean run that simply found nothing → `ok:true`, `text:""`, exit `0`.
* **stdout hygiene:** Trafilatura/lxml log lines and warnings are routed to stderr; stdout is exclusively the one JSON object.

**Pinning:** committed fully-pinned `requirements.txt` (full transitive freeze of a clean venv, exact `==`). README records the exact Python version and the one-command reproduce-from-lock and smoke-test steps.

## 6. Readability-JS adapter — `benchmark/oracles/readability-js/run.mjs`

**Responsibility:** read bytes → build a DOM with jsdom → run `@mozilla/readability` → map to the contract → print one JSON object.

**Design constraints:**

* **jsdom must be inert and offline.** Construct jsdom so that scripts never run and no resources are ever loaded: do **not** enable `runScripts`, do **not** install a `resources` loader, and attach a `VirtualConsole` that forwards to **stderr only** (never to process stdout). This gives no-network, no-JS-execution, and deterministic parsing for free. `--base-url`, when present, is passed as jsdom's document URL (so canonical/relative resolution works); when absent, jsdom's default is used and `canonical_url` may be `null`.
* **Readability is the extractor; the adapter does no content heuristics of its own.** It only reads trivial declared metadata from the DOM where the contract needs a field Readability does not return.
* **Field mapping:**
  * `oracle` = `"readability-js"`; `oracle_version` = the installed `@mozilla/readability` version, read at runtime from the package.
  * `title`, `text` = Readability's article title and its `textContent`, as-is. `text` is the primary comparison surface.
  * `html` = Readability's content HTML (`.content`); it is produced for free, so it is populated (unlike the Trafilatura side — accepted asymmetry per brief).
  * `word_count` = recomputed from `text` by §4 (NOT any library count).
  * `canonical_url` = `<link rel="canonical">` href read from the DOM, resolved against `--base-url`/document URL when relative; else `null`. (Readability's result does not carry canonical, so the adapter reads this one declared value directly — minimal, deterministic, symmetric in intent with the Trafilatura side.)
  * `language` = best-effort declared language (`<html lang>`), else `null`.
  * **Readability returning `null`** (no content found) is **not** an error: `ok:true`, `text:""`, exit `0`.
* **Error handling:** unreadable path / jsdom or Readability throwing / module load failure the script can catch → full JSON envelope with `ok:false`, `error`, non-zero exit.
* **stdout hygiene:** jsdom CSS/parse warnings and any console output go to stderr via the `VirtualConsole`; stdout is exclusively the one JSON object.

**Pinning:** committed `package.json` pinning **both** `@mozilla/readability` and `jsdom` to exact versions, plus committed `package-lock.json` (full closure). README records the exact Node version and the one-command reproduce-from-lock and smoke-test steps.

## 7. Reproducibility & pinning strategy

Unpinned oracles make the entire differential signal non-reproducible. Floor (mandated) for v1:

* **Trafilatura:** fully-pinned `requirements.txt` (full transitive freeze, exact `==`), committed. Python version recorded in README. `oracle_version` emitted at runtime.
* **Readability-JS:** exact `@mozilla/readability` **and** `jsdom` pins in `package.json`; `package-lock.json` committed. Node version recorded in README. `oracle_version` emitted at runtime.
* **Pin choice:** latest stable of each as of the build date (2026-05-17), recorded in the adapter README with a one-line note confirming there is no reason to prefer an older known-good.
* **Each README** states the exact upstream version represented and the reproduce-from-lock and smoke-test commands.

**Containerisation is deferred, not adopted.** Per the brief it is the recommended stretch, not mandated. Given the user's direction toward simplicity and a dev sandbox, v1 is **document-and-lock only**. A `Dockerfile` per oracle can be layered on later with zero contract change (the contract is the seam; the environment behind it is swappable). This is recorded as a deliberate decision (§10), not an omission.

## 8. The conformance harness — "the oracle of the oracles"

The brief lists this as a recommended stretch; this design **adopts it as part of "done"**, because it is what converts "the contract is frozen" from hope to fact and it is cheap.

**Form: a Rust integration test at `tests/oracle_contract.rs`, run by `cargo test`.** Rationale:

* This is a Rust repo; `cargo test` is the one canonical quality gate already implied by the grandparent brief's CI.
* It exercises the *exact subprocess seam* the downstream Rust runner will use — so this test is scaffolding that later clicks straight into the real runner (two jigsaw pieces from opposite sides), not throwaway.
* It is language-neutral between the two adapters (an external arbiter shelling out to both), which a Python- or Node-hosted harness would not be.
* No new language, tool, env var, or feature flag is introduced — consistent with the user's constraints.

**Rejected alternative:** a standalone Python/shell script under `benchmark/oracles/`. Rejected because it adds a third coordination point and biases the arbiter toward one adapter's ecosystem, for no benefit over a `cargo test`.

**Prerequisite, stated honestly (no flag):** the test requires Python and Node on PATH. It is **not** gated behind a feature flag or env var (the user asked to avoid these, and gating would hide a missing toolchain). At this milestone the oracles *are* the deliverable, so if a toolchain is absent the test **fails loudly** — that is the correct, honest behaviour, documented as a prerequisite in the repo README.

**Synthetic, committed fixtures** at `benchmark/oracles/fixtures/` — distinct from the real downstream corpus (which is a separate, out-of-scope stream). Synthetic = licence-clean, stable, intent-explicit. Minimal set:

| Fixture | Exercises |
|---|---|
| `example_com.html` | trivial content page → `ok:true`, non-empty `text` |
| `article.html` | clear synthetic article → both adapters extract substantive `text` |
| `empty.html` | head/script only, no body prose → the **Bug E2 guard**: `ok:true`, `text:""`, exit `0` |
| `malformed.html` | broken/unbalanced tags → adapter stays robust; defined `ok` semantics |
| `whitespace_stress.html` | body engineered so extracted `text` carries NBSP/NEL/em-space/ideographic-space/U+FEFF → word-count parity (§4.3) |

**What the harness asserts, for each adapter × fixture:**

1. stdout is **exactly one** JSON object that **validates against `contract.schema.json`**.
2. `ok` / `error` / exit-code semantics per §3.3 (including the `empty.html` Bug E2 guard and the `article.html` non-empty-`text` expectation).
3. **Determinism:** run twice, assert **byte-identical** stdout.
4. **Word-count parity:** on `whitespace_stress.html`, both adapters' `word_count` are equal to each other and to the `WORDCOUNT.md` expected value.

## 10. Decisions & rationale

| # | Decision | Rationale |
|---|---|---|
| D1 | Contract is a committed **JSON Schema file**, not prose | Turns "frozen" into a machine-checkable claim; schema is data, not the "shared code" the brief forbids |
| D2 | Word-count rule = **explicit 25 code-point set**, counted as `[^S]+` runs, embedded in both adapters | Bare `\s` differs across Python/JS *and* from Unicode `White_Space`; explicit set is the only byte-identical option; necessary duplication since no shared code |
| D3 | **Document-and-lock**, containerisation deferred | Brief default + user's simplicity direction + dev sandbox; container can be added later with zero contract change |
| D4 | Conformance harness = **Rust `cargo test`** | One canonical gate; neutral arbiter between adapters; prototypes the real runner's subprocess seam; no new tool/flag |
| D5 | Toolchain absence → **hard test failure**, no flag/env-var gate | User asked to avoid flags/env vars; at this milestone the oracles *are* the deliverable, so silence would be dishonest |
| D6 | **Full transitive freeze**, incl. Trafilatura's fallback closure | Barbaresi-benchmarked SOTA *is* the fallback cascade; pinning the package alone under-specifies the algorithm |
| D7 | `language` best-effort; **no langid dependency** added | Explicitly low-priority/null-acceptable; avoids a heavy non-deterministic dep; keeps closure small |
| D8 | `html` asymmetry tolerated (JS populates, Trafilatura may `null`) | Brief: `html` is not a comparison surface and not v1-gating; faithfulness beats forced symmetry here |
| D9 | Failure path **still emits the full JSON envelope** on stdout *and* a non-zero exit | Dual signalling: runner parses one uniform shape always, yet can still detect a crash via exit code |
| D10 | Conformance fixtures are **synthetic & committed**, separate from the real corpus | Licence-clean, stable, intent-explicit; the 50+ URL corpus is a separate out-of-scope stream |

## 12. Risks

* **R1 — Trafilatura non-determinism via ambient config or optional deps.** Mitigated by explicit committed config (no ambient), full transitive freeze, and the determinism assertion in the harness. *Residual:* a future Trafilatura release changes default behaviour — caught as a baseline diff via `oracle_version` + committed baselines.
* **R2 — Word-count drift between the two embedded copies.** Mitigated by `WORDCOUNT.md` as single source of truth and the end-to-end parity fixture (§4.3) that fails the build on any divergence.
* **R3 — jsdom emitting to stdout** (parser/CSS noise) corrupting the single-JSON-object guarantee. Mitigated by the mandatory stderr-only `VirtualConsole` and the harness's "exactly one valid JSON object" assertion.
* **R4 — Toolchain absent in CI** makes the harness fail. By design (D5) this is loud, not hidden. *Action:* document Python+Node as repo prerequisites for this milestone.
* **R5 — Library API differs from design-level expectations** for the pinned version. Mitigated by treating §5/§6 as constraints, not a verified call sequence: exact calls are written and verified against the pinned versions at build time, with the conformance harness as the acceptance gate.
