## 1. Purpose

Produce the project's first deliverable: a harness that, for each URL in a fixed corpus, runs the two oracle adapters and the `mdrcel` crate, scores their outputs against the locked oracle hierarchy, and emits a markdown report plus a regression check against a committed baseline. At Milestone 1 the crate returns "not implemented" for every URL — that is the intended floor. The harness is the instrument that makes "correct" measurable for a heuristic problem with no formal spec.

The harness is **scaffolding** (per the quality philosophy): critical to producing the building, not part of it. It is not shipped to crates.io. Its testable core is a handful of pure functions; everything else is process orchestration and file I/O.

## 2. Locked decisions

1. **Asymmetric oracle hierarchy** — supersedes the parent brief's "union of agreement":
   ```
   gold set  >  Trafilatura (#1)  >  Readability-JS (#2)
   ```
2. **Trafilatura is #1** — on a disagreement with no gold-set entry, Trafilatura's output is the target.
3. **Readability-JS is #2, a guardrail** — it does not win ties; it raises a flag when it extracts substantially more than Trafilatura on a structured page (suspected Trafilatura truncation).
4. **The gold set is not an oracle** — hand-decided ground truth for ~6–10 the consumer-critical / SEC-EDGAR URLs, authoritative only on those URLs (where Trafilatura is not benchmark-validated).
5. **Honest ceiling** — outside the gold set, mdrcel's measurable quality is capped at Trafilatura's. "Exceeds SOTA" is only provable on the gold set. The harness must not report divergence-from-Trafilatura as "improvement" anywhere except the gold set.
6. **Baseline-first** — the crate outputs "not implemented" initially; the first committed report is the floor.
7. **Gold-set freeze (methodological independence).** The thing-under-test team (#1) also owns the gold set, so: gold-set expected texts are curated and **committed before any crate tuning begins**; any post-freeze change requires explicit written justification in the commit message. This closes the one independence gap the oracle split does not cover.
8. **Team split & orchestration boundary (locked; consistent with sibling §2).** Team #1 = crate + harness; Team #2 = oracle adapters. The harness *orchestrates*: it spawns the two oracle subprocesses and calls the crate in-process. Team #2 owns the adapters being correct/pinned/independently runnable; it does not drive the integrated pass.
9. **Single-host reproducibility (accepted; container deferred, no Docker for v1).** Pinning cannot pin native `libxml2`/ICU, so cross-machine byte-identity is impossible without a container. v1 accepts this: the committed baseline is valid **only on one named host**, which is named in the `benchmark` README and stamped into the baseline file's metadata; runs on any other machine are advisory, not regression-gating.

## 3. Design constraints (from CLAUDE.md and Arthur's steer)

- **Simplicity over cleverness.** Smallest thing that works. If the architecture doesn't fit in your head, it's wrong.
- **No environment variables. No feature flags.** Convention over configuration. (The sibling adapter HLD has one tightly-scoped internal re-exec sentinel in *its* tree; the harness tree has none.)
- **No migration / rollback / backwards-compat machinery.** Development sandbox.
- **No premature abstraction.** Exactly **two** oracles, hardcoded by name. No plugin/registry/"extensible backend" — the M8-ring-road antipattern the parent brief warns against.
- **One language for the harness: Rust.** Deliberate deviation from the parent brief's `compare/report.py` sketch — see §10.
- **File-based artifacts only.** No database, no service. Immutable committed snapshots; baseline is a committed file.
- **Sync, no async.** Subprocess + file I/O; no runtime.

## 4. Architecture

### 4.1 Workspace structure (minimal change to current repo)

The repo today is a single crate (`mdrcel` at root). Add a Cargo **workspace** so the harness is a separate crate that depends on the library, but the library never depends on the harness:

```
mdrcel/
├── .gitattributes          # repo root — CONTENT owned by sibling Oracle HLD (§8 there);
│                           #   Team #1 ACTIONS its commit first in M0 (§12 step 0).
├── .gitignore              # repo root — owned by THIS HLD
├── Cargo.toml              # add [workspace] members = ["benchmark"]; [package] mdrcel stays at root
├── src/                    # the mdrcel library (unchanged location)
├── benchmark/
│   ├── Cargo.toml          # bin crate "benchmark"; path-deps mdrcel; harness-only deps live here
│   ├── README.md           # names the declared reproducibility host (§2.9)
│   ├── src/
│   │   ├── main.rs         # CLI entrypoint: `cargo run -p benchmark`
│   │   ├── corpus.rs       # load urls list, resolve snapshot paths, hashing
│   │   ├── oracle.rs       # invoke the two oracle subprocesses, parse contract JSON
│   │   ├── crate_run.rs    # call mdrcel::extract in-process
│   │   ├── metrics.rs      # PURE: tokenization, Jaccard, normalised edit distance
│   │   ├── score.rs        # apply the hierarchy → coverage/precision/agreement
│   │   ├── report.rs       # emit report.md + results.json
│   │   └── regression.rs   # diff current results vs committed baseline
│   ├── corpus/
│   │   ├── urls.tsv        # the corpus manifest (see §6)
│   │   ├── snapshots/      # <url-hash>.html — immutable, committed, dated in urls.tsv
│   │   └── gold/           # <url-hash>.txt + gold.tsv — hand-curated truth (see §7)
│   ├── oracles/            # OWNED BY THE ORACLE TEAM (incl. contract.schema.json,
│   │   │                   #   .gitignore, fixtures). Harness only invokes the entrypoints.
│   │   ├── contract.schema.json   # NORMATIVE contract artifact (oracle-owned)
│   │   ├── trafilatura/run.py
│   │   └── readability-js/run.mjs
│   ├── baseline/
│   │   └── results.json    # committed; regression compares against this; metadata names the host
│   └── runs/
│       └── <UTC-timestamp>/{report.md,results.json}   # per-run output (gitignored)
```

Rationale: keeps the library's dependency graph tiny (parent brief: `scraper`, `html5ever`, `regex`, `thiserror`); harness deps (`serde`/`serde_json`, `std::process`) are quarantined in `benchmark/`. No feature flags needed — a workspace member separates them structurally. Ownership map matches sibling §8: this HLD owns `benchmark/Cargo.toml`, the workspace, repo-root `.gitignore`; the oracle HLD owns `benchmark/oracles/**` and the *content* of repo-root `.gitattributes`.

### 4.2 Data flow (one run)

```
urls.tsv ──► for each URL:
   snapshot.html ──┬─► subprocess: oracles/trafilatura/run.py    ─► JSON ─┐
                   ├─► subprocess: oracles/readability-js/run.mjs ─► JSON ─┤
                   └─► in-process: mdrcel::extract(html, base_url) ───────┤
                                                                          ▼
                                                            normalise + tokenize (one impl)
                                                                          ▼
                                                   score against hierarchy (gold? → Traf → Read)
                                                                          ▼
                              results.json  ──►  report.md  ──►  regression vs baseline/results.json
```

The crate is called **in-process** (it's a Rust workspace dep). Only the two oracles are subprocesses (different runtimes). This asymmetry is intentional and simpler than uniform subprocessing.

## 5. The oracle contract — harness (consumer) side

**Normative source:** `benchmark/oracles/contract.schema.json` (oracle-owned). The field list below is a **non-normative restatement**; on any discrepancy the schema governs. The harness's serde types are validated against the committed schema in tests, not hand-maintained against this prose.

**Invocation (fixed convention, no configuration):**
```
python  benchmark/oracles/trafilatura/run.py    <abs-snapshot-path> [--base-url <URL>]
node    benchmark/oracles/readability-js/run.mjs <abs-snapshot-path> [--base-url <URL>]
```
Interpreter literals are bare `python` / `node`, exactly as the sibling expects. The Trafilatura adapter self-corrects its interpreter by construction (sibling §4 venv re-exec); the Node side has **no equivalent self-correction** — see open item §11.1 (routed to the oracle team).

**Output: a single JSON object on stdout.** Fields the harness reads (non-normative restatement of the schema):
```
oracle          : "trafilatura" | "readability-js"
oracle_version  : string | null     # null iff the library could not be imported
title           : string | null
text            : string            # NEVER null; "" when none; the primary comparison surface
canonical_url   : string | null
language        : string | null
ok              : bool
error           : string | null
```
- **`oracle_version` is `Option<String>`** in the harness's type — a failed import still emits a valid envelope with `oracle_version: null`. Typing it non-null would be a Bug-E2-class trap (a valid failure report rejected as unparseable).
- **The harness must NOT use `deny_unknown_fields`.** The oracle emits 10 fields; the harness reads 8. `html` and the oracle's own `word_count` are deliberately ignored — the harness recomputes word count itself (§8), so there is exactly one word-count implementation.
- **`text` is guaranteed valid UTF-8 before serialization** by the adapter (sibling §3.3), so `serde_json` cannot reject an otherwise-valid extraction.
- **Tri-state (sibling §3.4), consumer mapping:**
  - exit 0 + `ok:true` → `ok` (even if `text:""` — "found little" is a valid result, **not** an error).
  - non-zero or `ok:false` (they co-occur) or no parseable stdout → `oracle_error`. Never silently treated as empty content (the Bug E2 lesson, consumer side).
- **Timeout is a distinct status, not `oracle_error`.** A subprocess exceeding a fixed **180 s** wall-clock is recorded as **`oracle_timeout`**, a first-class status separate from `oracle_error`. Folding a legitimately-slow large SEC filing into `oracle_error` would be a Bug-E2-class conflation on exactly the named acceptance-critical cases. 180 s is a documented code constant, revisitable once real corpus timings exist (evidence-driven, not predicted).

**The crate adapter** (`crate_run.rs`) calls `mdrcel::extract`. At M1 the library returns a dedicated `ExtractError::NotImplemented`; the harness maps that to a first-class `not_implemented` status (distinct from `crate_error` and from `ok` with empty text).

## 6. Corpus

`benchmark/corpus/urls.tsv` — tab-separated, one row per URL, the manifest:

```
url<TAB>shape_class<TAB>snapshot_filename<TAB>fetched_date<TAB>note
```
- `shape_class` ∈ {wikipedia, sec_edgar, regulator, news, tech_blog, hub_index, edge_case} — drives the report's per-class breakdown.
- `snapshot_filename` = `<sha256(url)[:16]>.html`, stored under `corpus/snapshots/`, **committed and immutable**. Re-fetching a URL means a new row + new snapshot, never mutating an old one.
- Snapshots are produced by a **one-shot, out-of-band fetch step** (`cargo run -p benchmark -- fetch <url>`), not part of the scoring run. The scoring run never touches the network. This keeps runs deterministic and the hot path pure.

≥ 50 URLs spanning the shape classes in the parent brief §"What done means". The exact list is corpus curation work, tracked separately; this HLD fixes the *format and immutability rule*, not the contents.

## 7. Gold set

`benchmark/corpus/gold/` holds hand-curated ground truth for the ~6–10 URLs where neither oracle is trustworthy:

- `gold/gold.tsv`: `url<TAB>snapshot_filename<TAB>expected_text_file<TAB>min_words<TAB>max_words<TAB>why_critical`
- `gold/<url-hash>.txt`: the hand-extracted expected body text (plain text, same tokenization target as everything else).

**Freeze rule (locked, §2.7):** every gold entry is curated and committed **before any crate tuning begins**. After that freeze, changing a gold expected text or word band requires an explicit written justification in the commit message. This is the enforcement mechanism for gold-set independence given Team #1 owns both the crate and the gold set.

**Initial gold members** (from the parent brief's the named acceptance + the EDGAR gap):
1. Apple — English Wikipedia (≥ ~2 500 words expected)
2. Wm Morrison — English Wikipedia (≥ ~1 500 words expected)
3. SEC 10-K — `aapl-20250927.htm` (**word target re-derived from the document itself**, not any prior extractor's output — see §11.3)
4. `example.com` (body text only; no head/script metadata)
5. One older table-heavy SEC filing (EDGAR weak point)
6. One regulator publication (gov.uk or FCA, table-heavy)
7. One hub/index page — expected result is "classified as navigation, near-empty body"

For a gold URL, the gold text is the sole reference: Coverage/Precision are computed against it; Trafilatura and Readability are still recorded for context but do **not** define truth there.

## 8. Metrics (pure, in `metrics.rs` — the testable core)

**Single tokenizer, single word count.** `tokens(text)` = split on Unicode `\p{White_Space}`, drop empties, lowercase, NFC-normalise. `word_count` = `tokens(text).len()`. Applied identically to oracle `text`, crate `text`, and gold text. The harness never trusts an external word count. This one function is the comparability guarantee. (The set-based tokenizer is also why the adapters' per-language U+FFFD-count divergence, sibling §3.3, is benign here — an explicitly accepted assumption shared across both HLDs.)

**Primary metric — token-set Jaccard.** For sets A, B of tokens: `J = |A ∩ B| / |A ∪ B|`. Interpretable as "what fraction of the combined vocabulary do they share." Robust to reordering, harsh on omission/boilerplate inclusion — exactly the failure modes we care about.

**Secondary metric — normalised Levenshtein over the token sequence**, `1 - lev(seq_a, seq_b) / max(len_a, len_b)`. Captures structural/ordering differences Jaccard misses. Reported alongside, not used for pass/fail.

(Decision: token-set Jaccard primary + normalised edit distance secondary. simhash considered and rejected for v1 — extra dependency + tuning parameter for a signal edit distance already gives interpretably. Revisit only on a concrete need.)

Derived, per URL, **against the hierarchy**:
- **Coverage** = Jaccard(crate_tokens, reference_tokens) where `reference` = gold if present, else Trafilatura.
- **Precision proxy** = |crate ∩ reference| / |crate| — fraction of extracted tokens in the reference (penalises boilerplate inclusion).
- **Guardrail flag** = set when `word_count(Readability) > word_count(Trafilatura) × 1.25` on a non-hub page → "suspected Trafilatura truncation; needs gold adjudication." Documented code constant.
- **Agreement-on-disagreement** = where Trafilatura and Readability are far apart (Jaccard < 0.5), which one the crate is closer to. Distribution reported, not scored.

All metric functions are pure → unit + property tested: `J(x,x)=1`, `J(x,∅)=0`, symmetry, `0 ≤ J ≤ 1`; edit-distance identity/symmetry/triangle sanity. These properties are the harness's own testing oracle.

## 9. Report & regression

**`runs/<UTC-timestamp>/results.json`** — machine-readable: per-URL `{url, shape_class, statuses, word_counts, jaccard, edit_sim, coverage, precision, flags}` plus a run header recording the **host identity** (§2.9). The single source of truth for a run.

**`runs/<UTC-timestamp>/report.md`** — human-readable, generated from `results.json`:
1. Summary: corpus size, per-status counts (`ok` / `oracle_error` / `oracle_timeout` / `not_implemented` / `crate_error`), mean Coverage & Precision overall and per shape_class.
2. Coverage table: URL × {Trafilatura wc, Readability wc, crate wc, Coverage, Precision, flags}.
3. Disagreements ranked by severity: high Trafilatura↔Readability divergence **and** low crate-to-reference similarity = candidate bug, worst first.
4. Guardrail-flagged URLs (suspected Trafilatura truncation) — the gold-set candidate queue.
5. Gold-set section: pass/fail vs `min_words..max_words` and Coverage vs gold text.

**Regression** (`regression.rs`): compare `runs/<latest>/results.json` against committed `baseline/results.json`. Flag any URL where crate `word_count` shrank, or Coverage-to-reference dropped, by more than a documented constant (proposed 5%). Output a `REGRESSIONS` block at the top of `report.md`; exit non-zero if any regression (CI fails).

**Host pinning (§2.9):** the baseline is valid **only on the host named in its metadata and the `benchmark` README**. The regression check compares the running host against that name; on a mismatch it prints `BASELINE ADVISORY (host X, expected Y) — not regression-gating` and does **not** exit non-zero. CI runs on the declared host; everywhere else regression output is advisory. **Updating the baseline is a deliberate manual act**: copy `runs/<latest>/results.json` over `baseline/results.json` and commit, with the message stating why. No tooling, no migration.

## 10. Deliberate deviations & things we are NOT building

- **Report in Rust, not Python (deviation from parent brief's `report.py`).** The harness is already Rust; the report is data-shaping + markdown emission. A second language in the *harness* means a second toolchain/lockfile/failure-surface for zero gain. Python stays *inside the Trafilatura oracle* (other team, isolated).
- **No oracle plugin/registry abstraction.** Exactly two oracles, named in code.
- **No config file, no env vars, no flags.** Corpus path, oracle entrypoints, 180 s timeout, 5% regression threshold, 1.25 guardrail ratio are conventions/constants documented in the `benchmark` README.
- **No DB / no run history service.** `runs/` is gitignored scratch; `baseline/results.json` is the only persisted state, in git.
- **No network in the scoring path.** Fetch is a separate one-shot subcommand; snapshots are immutable committed inputs.
- **No retry/backoff on oracle subprocesses.** A failing oracle is a recorded `oracle_error`; a slow one `oracle_timeout`. Surfaced loudly, not papered over.
- **No Docker / container for v1.** Single-host reproducibility accepted (§2.9); the container is the deferred true fix and slots in later with zero contract change.

## 11. Open points / coordination

1. **Node version self-correction (routed to the oracle team).** The harness invokes bare `node`; the Readability adapter has no equivalent of the Trafilatura venv re-exec, so if the host `node` ≠ the pinned `.nvmrc` it runs unpinned and silently `ok:true`-wrong. Recommended fix (oracle side): the adapter asserts `process.version` against its pin → emits the §3.4 failure envelope on mismatch. Tracked cross-team; does not block harness M0 (harness only invokes `node`).
2. **Contract sign-off — DONE (2026-05-17).** Reconciled with the sibling HLD: schema-as-normative; `oracle_version` `Option`; no `deny_unknown_fields`; distinct `oracle_timeout`; tri-state mapped; UTF-8 guarantee relied upon. No open conflict remains; `contract.schema.json` is the single shared artifact and the harness validates against it.
3. **SEC 10-K gold target must be re-derived from the document**, not any prior extractor's output words. Gold-set curation work; the HLD only fixes that the target is document-derived.
4. **Corpus contents** (≥50 URLs) and **final gold list** are curation tasks tracked separately; this HLD fixes formats, immutability, the freeze rule, and the hierarchy maths — deliberately not the contents (evidence-driven, not predicted).
5. **Ownership/cadence — resolved.** This session carries the harness build through M0→M1; docs are written cold-start-robust; coordination with the oracle team is only at the §5 contract seam (now closed) and open item §11.1.

## 12. Build order (this deliverable)

0. **`.gitattributes` first (cross-team ordering, sibling §2.1).** Team #1 commits first in M0, so Team #1 actions the commit of the oracle-HLD-owned `.gitattributes` content (`* text=auto eol=lf`; fixtures/locks/schema `-text`) **and runs `git add --renormalize . && git commit` before any byte-stable artifact** (Cargo.lock, snapshot, fixture) is added. Skipping this corrupts byte-stability irreversibly.
1. Workspace + `benchmark` crate skeleton; `cargo run -p benchmark` prints "no corpus". Repo-root `.gitignore` (`runs/`).
2. `metrics.rs` + property tests — pure core first, TDD (the harness's own oracle). **No cross-team dependency.**
3. Corpus loader + 3 sample snapshots + `fetch` subcommand. **No cross-team dependency.**
4. Oracle invoker + serde types validated against `benchmark/oracles/contract.schema.json`. **Depends on the oracle team having committed `contract.schema.json`** (not the full adapters — just the schema).
5. Crate-in-process adapter returning `not_implemented`.
6. Scoring (hierarchy) → `results.json` (with host metadata).
7. `report.md` generation.
8. Regression vs committed baseline (host-pinned); commit the first baseline = the documented floor.

Steps 1–3 have no cross-team dependency and may proceed immediately. Step 4 is the only gate on the oracle team, and only on their `contract.schema.json`, not their full adapter build.
