## TL;DR

metadata subsystem to byte-parity with Python, and unmask the `<teiHeader>` incrementally — field by
field, fixture by fixture — as parity is reached.** Every unmask is a permanent, gated parity win.

## 2. The oracle, precisely

The TEI gate (`tests/trafilatura_tei_gate.rs`) runs, for each of 51 corpus snapshots:
`mdrcel::extract_to_tei(html, base_url, opts)` vs the Python oracle
(`benchmark/oracles/trafilatura/run.py --xmltei`), NFC-normalises both, and byte-compares.

- **`with_metadata` forcing:** `core.py` sets `with_metadata = True` whenever
  `output_format == "xmltei"` (settings.py ~144–149). So even though `run.py --xmltei` passes
  `with_metadata=False`, Python emits a fully-populated `<teiHeader>` (title/author/publisher/date/
  url/etc.). mdrcel's `extract_to_tei` (`src/lib.rs`) mirrors this — it **always** extracts metadata.
- **Today's mask:** the gate collapses `<teiHeader>…</teiHeader>` → `<teiHeader/>` on **both** sides
  before comparing, and shape-checks (not value-checks) the `<note type="fingerprint">`. So the gate
  currently proves only that the **`<text>` body subtree** is byte-identical (which it is, on all 39
  substantive fixtures). The header is unverified.
- **Accounting invariant:** `substantive + allowlist + deferred + fingerprint == 51`. Today
  `39 + 11 + 1 = 51`. The invariant assert must always hold; never let a fixture fall out of all
  buckets.

**Your unmask loop:** progressively narrow the neutralisation so more of the `<teiHeader>` is
byte-compared, fix mdrcel until it matches Python, and move fixtures from "header-masked" toward
"header-substantive." The end state is a gate that compares the *full* document (header + body) with
only the two principled masks in §3 remaining.

## 3. Scope

### In scope — drive these to byte-parity
The known weak areas (observed in M7, but **re-verify each against vendored source**):
- **Dates** (`<date>` in header): truncation, format, and *discovery* (which DOM node / meta tag /
  JSON-LD field Python picks). This is **htmldate parity** — `htmldate==1.9.4` is the de-facto
  sub-oracle. Expect this to be the largest sub-task.
- **Hostname / URL** (`<idno>`/url fields): `www.`-prefix stripping and normalisation. This is
  **courlan parity** — `courlan==1.3.2` is the sub-oracle.
- **Author / sitename quality**: extraction and cleaning of `<author>` and publisher/sitename.
- **`filedate`**: M4 Stage 6 deferred this — mdrcel has *no slot* for it. Adding the slot may be
  required for full header parity; treat as a sub-task, version-bump as MINOR if it adds a public
  `Extracted`/metadata field.
- Any other `<teiHeader>` field that diverges once you start unmasking (categories, tags, license,
  description) — mdrcel already has additive `Extracted` fields for most of these (M3 §5.2).

### Out of scope — leave masked / deferred, do NOT solve here
- **The blake2b fingerprint** (`<note type="fingerprint">`). mdrcel deliberately uses an FNV-1a
  substitute to avoid a crypto dependency, so it can *never* byte-match. **Keep it masked.** This is
  an Arthur-level dependency decision, documented at `wrk_docs/m7-deferred/fingerprint-blake2b.md`.
  Do not add a blake2b dependency.
- **U+2063 / `remove_control_characters`** (`wrk_docs/m7-deferred/507b9cdb.md`). Needs a
  Unicode-general-category facility not in the dep tree — another Arthur-level decision. If a fixture
  diverges *only* on a U+2063 leak, defer it (DEFERRED_KNOWN_DEFECT) with a note pointing at that ADR;
  do not try to solve it.
- **The other four gates** (txt/json/csv/xml). Those use `with_metadata=False` and must stay green.
  If your metadata changes touch shared code (`output.rs`, `metadata*.rs`) you **must** re-run all
  gates (§7) — but do not change those gates' masks or buckets.
- **Comments, dedup, favor_precision/recall, language filtering.** Separate future milestones.

## 4. Triage discipline (the 3-bucket rule)

Every fixture that doesn't byte-match falls into exactly one bucket, each requiring a written ADR:

1. **Substantive (target):** mdrcel was wrong, you fixed it, now byte-equal. No ADR; it just passes.
2. **`PYTHON_UNDER_EXTRACT_ALLOWLIST`:** Python is *wrong / non-spec* and mdrcel is correct. Requires
   an ADR in `wrk_docs/m8-allowlist/<hash>.md` proving, against vendored source, *why* Python is the
   wrong side. **This bar is high** — for metadata, prefer matching Python's behaviour (bugs and all,
   that is the definition of fidelity) unless Python literally crashes or emits malformed XML.
3. **`DEFERRED_KNOWN_DEFECT`:** mdrcel is wrong and the fix is blocked (e.g. blake2b, U+2063, or a
   dependency you're told not to add). Requires an ADR in `wrk_docs/m8-deferred/<hash>.md` stating the
   blocker and what would unblock it.

GREEN = no untriaged fixtures + the accounting invariant holds. A fixture sitting in neither a pass
nor a documented bucket is a **build failure**, by design.

## 5. Landmines (learned the hard way M5–M7)

- **The rcdom reparent-tail trap.** `markup5ever_rcdom` nodes are `Rc<Node>` with `RefCell` children;
  lxml `.tail` is the text *after* an element's close tag. `append`/`insert`/`extend` MOVE an element
  *with its tail*. This bug class has bitten 9 times. If you re-parent nodes while building the
  header, use the safe primitives `reparent_with_tail` / `insert_with_tail` in
  `src/readability/dom.rs` — do **not** hand-roll a move. (Less likely in metadata work than in
  serializers, but flagged because metadata extraction does walk/move DOM nodes.)
- **`xmltotxt` whitespace semantics** (HLD §2.1) — header text fields go through the same
  sanitisation; match it exactly.
- **NFC normalisation** — the gate compares NFC bytes; don't be surprised by composed/decomposed
  Unicode diffs in author names etc. The gate handles normalisation; your job is the *content*.
- **Don't tune to the corpus blindly.** Fixing one fixture's date by special-casing its DOM shape is
  anti-inversion. Find the *general* rule Python (htmldate/courlan) applies and port that.
