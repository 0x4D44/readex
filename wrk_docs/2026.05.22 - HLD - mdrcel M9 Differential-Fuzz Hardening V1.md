## 1. The design in one paragraph

Our fidelity is proven on **51 hand-curated snapshots**, which is structurally "testing on the
training set" — a strong claim about those 51 pages and a weak claim about page #52. M9 widens that
to **low-thousands of real-world pages from a Common Crawl slice**, runs the existing **one-reference
byte-diff** (mdrcel vs pinned Python `trafilatura`) at scale, and funnels the divergences through a
**mechanical-cluster-then-batch-LLM** triage pipeline into the existing 3-bucket discipline
(substantive / allowlist / deferred). The bulk corpus is **ephemeral** (reproducible from a manifest,
git-ignored); only **minimised reproducers of real bugs are promoted** into the committed gate
corpus as permanent regressions. The goal is to convert "parity on the corpus" into "parity on the
real distribution" and discover the unknown-unknowns 51 fixtures cannot show us. No new heavyweight
deps, no feature flags, no env vars: everything is invoked explicitly via CLI/test args.

---

## 2. Goal & non-goals

**Goal.** Find port-fidelity bugs (mdrcel diverging from Python trafilatura v2.0.0) that the curated
corpus cannot surface, triage them efficiently, fix the real ones, and lock each fix as a permanent
regression fixture.

**Non-goals (explicit, to keep the surface small):**
- **Not** a quality/benchmark exercise. The three-way comparison against `readability-js` answers a
  *different* question (which oracle is right about "main content") and belongs to a later
  quality/shipping track. M9 is **one-reference: mdrcel vs trafilatura only.**
- **Not** a metadata effort — that's M8. M9 stays out of `metadata*.rs` and the TEI header (see §9
  coordination); non-metadata divergences only.
- **Not** tens-of-thousands scale. We build for low-thousands and deliberately do **not** build a
  persistent oracle server, sharded storage, or distributed fetching. (M8-ring-road rule: don't build
  on-ramps to a road we haven't decided to build.)
- **Not** solving the known deferrals (blake2b fingerprint, U+2063). The pipeline *masks* them; it
  does not fix them.

---

## 3. Decisions locked

| Decision | Value | Rationale |
|---|---|---|
| Corpus source | Common Crawl slice | Realistic distribution; best unknown-unknown discovery |
| First-run size | Low thousands | Stresses fidelity + the triage pipeline without demanding server-grade oracle throughput |
| Reference count | One (mdrcel vs trafilatura) | Fidelity question; readability three-way is a separate track |
| Input strategy | Real-world harvest first; structure-aware mutation second; **no** pure-random | Bug-density per input; random produces trivial agree/parse-error noise |
| Triage | Mechanical cluster → minimise → **batch** LLM skill → supervisor ratifies | LLM cost scales with *distinct root causes*, not divergences |
| Bulk corpus persistence | Ephemeral; manifest-reproducible; git-ignored | Low-thousands of HTML pages is repo bloat and is reproducible |
| Permanent artifacts | Minimised reproducers of confirmed bugs, committed to gate corpus | Fuzzing *feeds* the permanent gate |

## 4. Pipeline & data flow

```
                                                       (slow, Python, run ONCE)
[CC WARC slice] --harvest--> fuzz_corpus/<sha>.html --batch oracle--> oracle_cache.jsonl
                              + manifest.jsonl                              |
                                                                           v
   (fast, Rust, re-runnable without Python)                      [diff + classify + cluster]
                                                                           |
                                                          divergences.jsonl + clusters.json
                                                                           |
                                            (per cluster representative only, re-invokes Python)
                                                                   [minimiser] --> min/<sig>.html
                                                                           |
                                                                           v
                                            [batch LLM triage skill]  (hypothesis + vendored-source
                                             verify + draft bucket/ADR; asymmetric trust per §7)
                                                                           |
                                       +-----------------------+-----------+--------------------+
                                       v                       v                                v
                                  substantive             deferred (draft)               allowlist (DRAFT —
                                  (fix mdrcel,             rubber-stamp                    SUPERVISOR RATIFIES,
                                   promote fixture)                                        never auto-commit)
```

**Key property:** the slow Python step (batch oracle) runs **once per corpus** and is cached. All
iterative work (re-clustering, signature tuning) reads `oracle_cache.jsonl` and never re-invokes
Python. Only the minimiser re-invokes Python, and only on the handful of cluster representatives.

## 5. Component design (maximal reuse, minimal new surface)

### 5.1 Shared comparison-normalisation module — *prerequisite refactor*
Today each gate file (`tests/trafilatura_*_gate.rs`) carries its **own copy** of NFC-normalisation +
fingerprint mask + U+2063 mask + (TEI) header mask. The fuzz diff MUST apply byte-identical masking
or every input trips known deferrals and the signal drowns.

**Change:** extract the canonical normalisation into **`tests/common/mod.rs`** (the idiomatic Rust
integration-test sharing pattern) exposing `normalize(output, format) -> String`. Refactor all gates
to `mod common;` and call it. **Invariant:** every gate byte-identical-green before and after the
refactor (this is a pure de-duplication; behaviour must not move).

The fuzz harness lives *in the same test crate* (§5.4), so it shares `common` directly — no
cross-crate plumbing, no new workspace member, no public-API widening.

### 5.2 Harvester — `benchmark/fuzz/harvest.py`
Downloads N Common Crawl WARC segment file(s) over HTTPS from `data.commoncrawl.org`, streams records
with `warcio`, keeps `text/html` responses decoded to UTF-8 that pass a cheap content filter (has
some `<p>`/body text; size between ~1KB and ~2MB), samples to the target count, and writes:
- `benchmark/fuzz_corpus/<sha256-prefix>.html` (the snapshot; git-ignored)
- appends `benchmark/fuzz/manifest.jsonl`: `{sha, source_url, warc_path, warc_offset, length}`
  (committed — makes the corpus reproducible without storing it)

**Language note:** Python is a *deliberate exception* to the Rust-default rule (CLAUDE.md "unless
otherwise specified"): WARC tooling (`warcio`) is Python-native and battle-tested, and this is
oracle-adjacent data-prep that runs rarely. The actual *project logic* (diff/cluster/minimise) stays
in Rust. `warcio` is added to the oracle venv's requirements, pinned.

**Network/licensing posture:** harvesting is a one-off network operation against public CC data,
whose Terms of Use permit research and derivative use. Bulk snapshots are git-ignored and ephemeral;
only the manifest and tiny minimised reproducers (§5.6) are committed — consistent with the existing
51 committed snapshots, but far smaller.

### 5.3 Batch oracle — extend `benchmark/oracles/trafilatura/run.py`
Add a `--batch <dir>` subcommand: one Python process, one `trafilatura` import, loops over every
`*.html` in the dir, emits one JSONL line per file `{sha, txt, markdown, json, csv, xml, xmltei}`
(all formats in one pass — cheap once parsed). This kills the per-doc subprocess-spawn cost, which is
the only real throughput problem at low-thousands. **No persistent server** — a single batch
invocation writing `oracle_cache.jsonl` is sufficient and far simpler.

### 5.4 Diff + classify + cluster — `tests/fuzz_diff.rs` (an `#[ignore]`d harness)
A normal integration test marked `#[ignore]`, invoked explicitly:
`cargo test --test fuzz_diff -- --ignored --nocapture`. (No feature flag, no env var — `#[ignore]`
*is* the opt-in mechanism Rust already provides.) For each corpus doc it:
1. runs mdrcel for each format, normalises both sides via `common::normalize` (§5.1), byte-compares;
2. on mismatch, computes a **signature** (§6) and a **shape-class**;
3. writes `benchmark/fuzz/divergences.jsonl` (per-doc) and `benchmark/fuzz/clusters.json`
   (signature → {count, representative sha, shape-class}).
It reads `oracle_cache.jsonl` for the Python side — Python is never spawned here.

### 5.5 Minimiser — `tests/fuzz_min.rs` (also `#[ignore]`d)
Delta-debug (ddmin) over the representative's HTML: repeatedly drop DOM subtrees / attributes / text
runs and re-test "still diverges with the *same signature*?" The interestingness test re-runs mdrcel
+ a *single-doc* Python oracle call (the one place the cache can't help, but it runs only on the
dozens of representatives, not the corpus). Emits `benchmark/fuzz/min/<signature>.html`.

### 5.6 Batch LLM triage skill — `.claude/skills/fuzz-triage/`
Per §7. Consumes the minimised representatives + their diffs, emits draft ADRs and a triage report.

### 5.7 Promotion
When a divergence is root-caused as a real mdrcel bug **and fixed**, its minimised reproducer is
copied into the committed gate corpus (`benchmark/corpus/snapshots/<sha>.html`) and the relevant
existing BLOCKER gate picks it up automatically. This is the only path by which fuzz output becomes
permanent. The 51→N growth of the committed corpus is therefore *bug-driven and minimal*, not bulk.

## 6. The clustering signature (the make-or-break, kept deliberately coarse v1)

Signature = `(format, shape_class, locus)` where:
- **shape_class** ∈ {whitespace-only, text-content, attribute, missing-element, extra-element,
  structural} — derived from a cheap structural compare of the two normalised outputs;
- **locus** = the tag-name (or serializer context) of the *first* differing node.

Start coarse on purpose: coarse over-merges (cheap, may hide a second cause in one cluster) are
**corrected by the LLM pass**, which flags "this cluster contains two distinct causes" and feeds the
signature refinement. Fine-from-the-start would splinter one bug across many clusters and waste LLM
budget. We tune the signature *from LLM feedback*, not up front. (CLAUDE.md: don't predict the
future; iterate.)

---

## 7. Triage skill contract — asymmetric trust + anti-inversion

The skill processes one cluster representative at a time and, for each, **forms a root-cause
hypothesis, then opens the vendored Python source
(`benchmark/oracles/trafilatura/.venv/Lib/site-packages/...`) and confirms it** before asserting
anything. It then recommends a bucket with trust that is *asymmetric by blast radius*:

| Bucket | Meaning | Autonomy |
|---|---|---|
| **substantive** | mdrcel wrong; fixable | Self-proving (fix turns the gate green). Skill may draft the fix; supervisor reviews the diff. |
| **deferred** | mdrcel wrong; blocked on a dependency we're told not to add | Low risk. Skill drafts ADR in `wrk_docs/m9-deferred/`; supervisor rubber-stamps. |
| **allowlist** | **the reference oracle is wrong, mdrcel is right** | Extraordinary claim — the bucket where you'd *hide your own bugs*. Skill drafts with vendored-source proof in `wrk_docs/m9-allowlist/`; **supervisor MUST ratify; never auto-committed.** |

This guardrail is the doctrine: an LLM is exactly the engine that produces a comfortable "Python is
wrong" narrative to make a divergence disappear. Auto-allowlisting would invert the oracle. The skill
*accelerates the legwork* (hypothesis + source-verification + draft); it does **not** get to declare
the reference wrong.

Triage runs as a **batch** pass (decided): let the mechanical layer finish and globally dedupe before
spending any LLM tokens, so we never triage a cluster a later input would have merged.

## 9. Coordination with M8 (parallel stream)

- **M9 owns:** everything under §8 "NEW", `run.py`'s `--batch`, and non-metadata divergence fixes
  (serializers, xpath, cleaning, tail-handling).
- **M9 does NOT touch:** `metadata.rs`, `metadata_url.rs`, `metadata_jsonld.rs`, the `<teiHeader>`
  rendering in `output.rs`, or the TEI gate's masks. If the fuzzer surfaces a metadata divergence,
  M9 **records it for M8** and does not fix it.
- **Shared seam = `tests/common/mod.rs` and `output.rs`.** The §5.1 normalisation refactor should
  land *early* and be announced, since both streams depend on stable gate behaviour. `output.rs`
  edits stay localised; commits atomic. The `m9-*` ADR trees are disjoint from `m8-*`, so no
  conflict.
- **Supervisor is the integration point:** main stays gate-green; commits serialised; no two agents
  edit `output.rs` simultaneously.

## 10. Simplicity ledger — what we deliberately did NOT build

- **No feature flags, no env vars.** Opt-in is `#[ignore]` + explicit `--ignored`; data-prep is a
  separate script. Configuration is CLI args, not ambient state.
- **No persistent oracle server / IPC.** A single cached batch invocation suffices at low-thousands.
- **No new workspace crate for test support.** `tests/common/mod.rs` is enough; no public-API
  widening of `mdrcel`.
- **No migration/rollback/back-compat.** Dev environment.
- **No bulk corpus in git.** Manifest + promoted minimal reproducers only.
- **No three-way (readability) comparison.** Different question, different track.
- **No ML/embedding clustering.** A cheap structural signature + LLM refinement beats an opaque
  vector clustering we couldn't explain or debug.

## 11. Staging plan (= the prerequisite critical path)

1. **Stage 1 — shared normalisation refactor** (§5.1). Gates byte-identical-green before/after. Land
   early; it's the M8-shared seam.
2. **Stage 2 — batch oracle** (`run.py --batch`, §5.3) + `warcio` pin. Validate on the existing 51.
3. **Stage 3 — harvester** (§5.2). Pull a first low-thousands slice; build manifest. *(Needs the
   network/licensing go-ahead — already granted: CC slice.)*
4. **Stage 4 — diff/classify/cluster** (`fuzz_diff.rs`, §5.4, §6). First clusters report.
5. **Stage 5 — minimiser** (`fuzz_min.rs`, §5.5).
6. **Stage 6 — triage skill** (`.claude/skills/fuzz-triage/`, §7). First batch triage.
7. **Stage 7 — fix + promote loop** (§5.7): work the substantive clusters, promote regressions,
   ratify allowlist entries, document deferrals. Iterate until the cluster backlog is drained or
   hits diminishing returns.

Stages 1–6 are the build; Stage 7 is the payoff and where the real fidelity wins land. The build
(1–6) is comfortably longer than M8's unmask grind — so M8 finishes before M9 runs "in anger," and
the M8/M9 metadata handoff seam never actually materialises.

## 12. Risks

- **Signature quality.** Too coarse hides bugs; too fine wastes budget. Mitigation: coarse-v1 +
  LLM-feedback refinement (§6). Accept that the first run's clustering is imperfect.
- **LLM anti-inversion drift.** Mitigated by the §7 asymmetric-trust guardrail (no auto-allowlist).
- **Encoding/charset noise.** Real-world HTML has mojibake; both sides must decode identically.
  Harvester decodes to UTF-8 once; mdrcel and the oracle both consume the same decoded bytes, so
  decoding divergence is removed as a variable (we test extraction fidelity, not charset detection —
  the latter is explicitly the caller's job per the founding brief).
- **Triage volume.** Even clustered, the first run could surface more real bugs than one stream can
  fix. Acceptable: Stage 7 is iterative; we fix highest-frequency clusters first and the rest are
  documented backlog, not blockers.
