### 1.1 Annotated behaviours

#### (a) Length cap тАФ `settings.py:295-296`

```python
if len(value) > 10000:
    value = value[:9999] + "тАж"
```

- **Trigger condition:** `len(value) > 10000` (strict `>`). A 10,000-char value
  is NOT truncated; 10,001-char is.
- **`len()` is Python char count**, not byte count. Each `str` codepoint counts
  as 1 regardless of UTF-8 encoded byte length. (Python `str` is conceptually a
  sequence of Unicode codepoints; `len(s)` returns the codepoint count.)
- **Truncation form:** `value[:9999]` takes the FIRST 9999 codepoints, then
  concatenates `"тАж"` тАФ that is **U+2026 HORIZONTAL ELLIPSIS** (a single
  character, NOT three ASCII dots). Verified empirically with the vendored
  Python interpreter (`Phase 2E probe ┬з0`):
  ```
  s = 'a' * 10001
  val = s[:9999] + 'тАж'
  len(val) == 10000
  hex(ord(val[-1])) == '0x2026'
  ```
- **Whitespace handling:** none тАФ the truncation is unconditional on the
  9999-codepoint boundary. If the codepoint at index 9998 is a combining mark
  or the lead-half of an emoji ZWJ sequence, Python ALSO cuts there. (Real
  metadata-slot values that approach 10K codepoints are virtually all
  long descriptions or accidentally-included page text; the policy is "cut
  at codepoint, append ellipsis, move on" and we mirror it.)
- **Final length:** exactly 10000 codepoints (9999 + 1 ellipsis).

#### (b) HTML unescape тАФ `settings.py:298` тЖТ CPython `html.__init__.py:122-132`

```python
def unescape(s):
    """
    Convert all named and numeric character references (e.g. &gt;, &#62;,
    &x3e;) in the string s to the corresponding unicode characters.
    This function uses the rules defined by the HTML 5 standard
    for both valid and invalid character references, and the list of
    HTML 5 named character references defined in html.entities.html5.
    """
    if '&' not in s:
        return s
    return _charref.sub(_replace_charref, s)
```

Vendored at `C:\Users\marti\AppData\Local\Python\pythoncore-3.14-64\Lib\html\__init__.py`
(located via `python -c "import html; print(html.__file__)"`). Three sub-pieces:

##### (b.i) The regex `_charref` (`html/__init__.py:118-120`)

```python
_charref = _re.compile(r'&(#[0-9]+;?'
                       r'|#[xX][0-9a-fA-F]+;?'
                       r'|[^\t\n\f <&#;]{1,32};?)')
```

- Matches `&` followed by a captured group that is EITHER:
  - decimal numeric `#[0-9]+;?` (optional trailing `;`), OR
  - hex numeric `#[xX][0-9a-fA-F]+;?`, OR
  - named: 1-32 chars from the set `[^\t\n\f <&#;]` (i.e. not TAB / LF / FF /
    SPACE / `<` / `&` / `#` / `;`), optional trailing `;`.
- The 32-char ceiling is set against the longest HTML5 named entity
  (`CounterClockwiseContourIntegral;` = 32 chars; verified by probe).
- The semicolon is OPTIONAL on the regex, which is how the ~106 legacy
  no-semicolon entities (e.g. `&amp`, `&AMP`, `&copy`, `&gt`) get matched.

##### (b.ii) Per-match handler `_replace_charref` (`html/__init__.py:91-115`)

```python
def _replace_charref(s):
    s = s.group(1)
    if s[0] == '#':
        # numeric charref
        if s[1] in 'xX':
            num = int(s[2:].rstrip(';'), 16)
        else:
            num = int(s[1:].rstrip(';'))
        if num in _invalid_charrefs:
            return _invalid_charrefs[num]
        if 0xD800 <= num <= 0xDFFF or num > 0x10FFFF:
            return 'я┐╜'
        if num in _invalid_codepoints:
            return ''
        return chr(num)
    else:
        # named charref
        if s in _html5:
            return _html5[s]
        # find the longest matching name (as defined by the standard)
        for x in range(len(s)-1, 1, -1):
            if s[:x] in _html5:
                return _html5[s[:x]] + s[x:]
        else:
            return '&' + s
```

Three numeric-charref special tables (full quotes in ┬з1.3 below):

- **`_invalid_charrefs`** (35 entries, `html/__init__.py:30-65`): the
  Windows-1252 substitutions plus a few oddballs. E.g. `0x80 тЖТ 'тВм'`
  (EURO SIGN), `0x0D тЖТ '\r'`, `0x85 тЖТ 'тАж'` (HORIZONTAL ELLIPSIS).
- **Surrogate guard:** `0xD800..=0xDFFF` and `>0x10FFFF` тЖТ `'я┐╜'`.
- **`_invalid_codepoints`** (~80 entries, `html/__init__.py:67-88`): codepoints
  that decode to **empty string** (`return ''`). Examples: `0x01..0x08`,
  `0x0B`, `0x0E..0x1F` (NUL-region controls except CR/LF/TAB/FF тАФ note `0x09`
  TAB, `0x0A` LF, `0x0C` FF NOT in invalid list because they're whitespace),
  `0x7F..0x9F`, non-characters `0xFDD0..0xFDEF`, plane-end non-characters
  `0xFFFE`, `0xFFFF`, `0x1FFFE`, тАж `0x10FFFE`, `0x10FFFF`.

Named-charref **longest-prefix matching:**

- If the captured name (e.g. `notreal;`) is not in `_html5`, peel one char
  off the right and try again, until length 2.
- E.g. `&notreal;` тЖТ captured `notreal;` тЖТ not in table тЖТ try `notreal`,
  `notrea`, `notre`, `notr`, `not` тЖТ `not` IS in `_html5` (decoding to
  `'┬м'` ┬м NOT SIGN) тЖТ return `'┬м' + 'real;'` = `'┬мreal;'`.
- If no prefix down to length 2 matches, return `'&' + s` (verbatim with
  the leading `&` re-inserted).

##### (b.iii) The named-entity table `_html5` (`html.entities.html5`)

Verified by `python -c "from html.entities import html5; print(len(html5))"`:

### 1.2 Order of operations тАФ verbatim quote settles it

From `settings.py:289-299`, the order INSIDE the `if isinstance(value, str):`
arm is **lexically literal**:

1. (`:295-296`) Length cap тАФ runs FIRST. The 9999-char slice + ellipsis is
   applied to the raw, still-escaped value.
2. (`:298`) `unescape(value)` тАФ runs SECOND, on the already-capped value.
3. (`:298`) `line_processing(...)` тАФ runs THIRD, wrapping the unescape output.
   `line_processing` itself internally does:
   1. `&#13;` / `&#10;` / `&nbsp;` entity-substitution (`utils.py:286`),
   2. `remove_control_characters` (Phase 1 port), then
   3. `LINES_TRIMMING` regex + `.strip()` (Phase 1 already routes through
      `clean_and_trim_metadata`'s `strip_control_chars`).

**Critical observation:** the cap-then-unescape order means a 9999-char
`&amp;`-bearing value, after capping, may have the trailing `&am` left as a
DANGLING entity prefix. `unescape` will then see `&am` (no semicolon, but
matches `_charref`'s `[^\t\n\f <&#;]{1,32};?` because `;?` is optional),
attempt longest-prefix match: `am` is NOT in `_html5`, so no decode тЖТ return
`&am`. So a tail-cut mid-entity leaks `&am` literally into the output.
**mdrcel must replicate this** тАФ DO NOT pre-decode then cap.

## 3. Algorithm specification

Pseudocode (NOT Rust) тАФ order strictly per Python `settings.py:295-298`:

```
function clean_and_trim_slot(v: String) -> String:
    # Step 1 тАФ length cap. Char count, not byte count.
    if char_count(v) > 10000:
        v = take_first_chars(v, 9999) + U+2026

    # Step 2 тАФ html.unescape. Pure entity decode; no whitespace change.
    v = html_unescape(v)

    # Step 3 тАФ line_processing. Already part of the Phase 1 path.
    v = strip_control_chars(v)
    # (NOTE: Phase 1 ports remove_control_characters only; the entity-sub
    #  pre-pass and LINES_TRIMMING/strip Phase-1 deferred outside of
    #  clean_and_trim_metadata's body тАФ see ┬з3.4 below.)

    return v
```

### 3.1 Step 1 тАФ length cap

- Trigger: `chars().count() > 10_000`.
- Truncation: take first 9999 codepoints (`chars().take(9999)`), collect to a
  `String`, then push the single `char` `'\u{2026}'` (HORIZONTAL ELLIPSIS).
- Result: a `String` of exactly 10000 codepoints when truncation fires.
- **Rust correctness note:** `String::len()` is byte length, NOT char count.
  Naively writing `if v.len() > 10000` would over-truncate on ASCII (correct
  byte-count = char count) and under-truncate on multi-byte UTF-8 (where 10k
  chars could be up to ~40k bytes). Use `v.chars().count()` for the trigger
  and `v.chars().take(9999).collect::<String>() + "\u{2026}"` (or push) for
  the cut.

### 3.2 Step 2 тАФ `html.unescape` (the meat of Phase 2E)

Faithful port of Python `html/__init__.py:91-132`. Pseudocode:

```
function html_unescape(s: String) -> String:
    if '&' not in s: return s           # python fast-path, html/__init__.py:130

    out = empty String
    i = 0
    while i < len(s):
        if s[i] != '&':
            push s[i]; i += 1
            continue

        # Try to match _charref starting at i. The regex is:
        #   &(#[0-9]+;?|#[xX][0-9a-fA-F]+;?|[^\t\n\f <&#;]{1,32};?)
        attempt = scan_entity_body(s, i+1)  # returns Option<(body, end_idx)>
        if attempt is None:
            push '&'; i += 1
            continue

        (body, end_idx) = attempt
        push replace_charref(body)
        i = end_idx

    return out

function scan_entity_body(s: &str, start: usize) -> Option<(String, usize)>:
    # Returns the captured-group body (without leading '&') and the byte
    # index just past the end of the match.
    # Three branches, matching Python's regex alternatives in order:

    if s.starts_with('#') at start:
        if s.starts_with('#x') or '#X' at start:
            # Hex numeric. Match #[xX][0-9a-fA-F]+;?
            scan hex digits (1..) then optional ';'.
            if no hex digit follows: return None
            ...
        else:
            # Decimal numeric. Match #[0-9]+;?
            scan dec digits (1..) then optional ';'.
            if no decimal follows: return None
            ...
    else:
        # Named. Match [^\t\n\f <&#;]{1,32};?
        scan up to 32 chars not in the excluded set, then optional ';'.
        if 0 chars matched: return None

    return Some((captured_body, byte_index_after_match))

function replace_charref(body: String) -> String:
    # Mirrors html/__init__.py:91-115
    if body[0] == '#':
        # Numeric.
        digits = body[1..].trim_trailing(';')
        if body[1] in 'xX':
            num = parse_u32(digits[1..], radix=16)
        else:
            num = parse_u32(digits, radix=10)

        if num in _invalid_charrefs:
            return _invalid_charrefs[num]   # 35-entry table
        if num >= 0xD800 and num <= 0xDFFF or num > 0x10FFFF:
            return "\u{FFFD}"
        if num in _invalid_codepoints:
            return ""                       # ~80-entry set; empty string
        return chr(num)
    else:
        # Named. body ends with ';' or not (legacy).
        if NAMED_ENTITIES.contains(body):
            (cp1, cp2) = NAMED_ENTITIES[body]
            return chr(cp1) + (chr(cp2) if cp2 != 0 else "")

        # Longest-prefix fallback: peel chars off the right, length down to 2.
        for x in range(len(body)-1, 1, -1):
            prefix = body[..x]
            if NAMED_ENTITIES.contains(prefix):
                (cp1, cp2) = NAMED_ENTITIES[prefix]
                return chr(cp1) + (chr(cp2) if cp2 != 0 else "") + body[x..]

        return '&' + body                   # python: return '&' + s
```

### 3.3 Step 3 тАФ strip (Phase 1, untouched)

### 3.4 What Phase 2E DOES NOT change

The two non-strip parts of Python's `line_processing` (`utils.py:286-289`) тАФ

**Verification:** Phase 1 HLD ┬з5 O1 resolution chose the

Phase 2E adds the cap and unescape per the literal Python source. The

/// Python `value[:9999] + "тАж"` cap (settings.py:295-296).
/// Char count, NOT byte count.
fn cap_at_python_length(v: &str) -> Cow<str> {
    // Fast path: count chars; if <= 10_000, return borrowed input.

fn clean_and_trim_metadata(m: &mut Metadata) {
    fn process_in_place(slot: &mut Option<String>) {
        if let Some(v) = slot.take() {
            // Step 1: 10_000-char cap (settings.py:295-296).

        }

    process_in_place(&mut m.title);
    process_in_place(&mut m.author);

}

## 7. Risk assessment

| Risk | Likelihood | Mitigation |
|---|---|---|
| **`web_atoms` table cardinality drift.** A future `web_atoms` minor-version bump changes the entity count and Python's table stays at 2231 тАФ silent fidelity drift. | LOW | The exact pin (`=0.2.4`) + the test row `unescape_longest_named_entity` (which decodes the longest-known entity) make any drift loud at upgrade time.
