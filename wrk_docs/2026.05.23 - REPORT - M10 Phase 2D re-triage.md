All four records share the same signature: mdrcel xml has `<p>\n      <td>…</td>\n      <td>…</td>\n  …`

**Hypothesis:** these are the exact same html5ever-vs-lxml table-parsing semantic
gap identified as Phase 2 Hypothesis C — `<td>`/`<th>` tokens encountered outside a

elements where they are. The rescue pass's NEW `<p>` wrappers (Phase 2C) now wrap
those stray `<td>`/`<th>` tokens that survived parse, exposing them in the xml
serialisation as element children of a `<p>` — a shape they didn't have in any
pre-2C records because pre-2C the leading div-text run kept these inline.

**Same parser-layer-deep root cause as Hypothesis C.** Cannot be patched without
parser changes. NOT a regression — the records were already divergent pre-2C; only
the shape signature changed.

## 5. Root cause hypotheses

### 5a. Hypothesis A (selection-phase, ~254 records, 57.7% post-2C) — UNCHANGED

Same as Phase 2 §3 hypothesis A. Cross-format same-sha correlation still holds
(81-82% G1↔G2 and G1↔xml). Selection algorithms remain algorithmically
equivalent — divergence is in HTML parsing (html5ever strict vs lxml
permissive). Phase-3 territory; multi-session refactor; no fix proposed here.

**Confidence:** DEFERRED.

### 5b. Hypothesis B (blank-line accounting, ~11 records md + 3 txt) — FALSIFIED already

Phase 2B falsified the original framing (the serialiser is faithful; the
upstream tree differs). The rescue pass took 42% out of the md group, leaving
the residue at 11 md + 3 txt = 14 records, still mixed direction (5 MORE / 3
FEWER / 3 SAME-MAX). No new framing surfaces in this re-triage.

**Confidence:** UNCHANGED MEDIUM. Locus is upstream of the serialiser
(tree-build / text-vs-tail apportionment); per-record snowflake; not a single
patch.

### 5c. Hypothesis C (SEO-spam tag leakage, ~21 records + 4 N1) — UNCHANGED

Same parser-layer-deep root cause as Phase 2 §3 hypothesis C. Records have not
moved meaningfully (pre-2C had ~47 spam-pattern records; post-2C has 21 +
4 = 25 — but the pre-2C count was per-record-with-spam-substring, and the
counting method here is slightly stricter so the apparent drop may be partly
methodological). The 4 N1 records are the same root pattern in a different
shape.

**Vendored Python source (re-cited for completeness):**
- `htmlprocessing.py:48-80` — `tree_cleaning`. Calls `strip_tags(tree,
  stripping_list)` then `delete_element(elem)` for each cleaning_list match.
- `settings.py:349-404` — `MANUALLY_CLEANED` includes `iframe`, `fieldset`,
  `rp`, `rt`, `rtc`, `source`.
- `settings.py:407-429` — `MANUALLY_STRIPPED` includes `tbody`, `tfoot`,
  `thead`, `ruby`, `cite`, `dfn`, `acronym`.

**mdrcel mirror:**
- `src/trafilatura/cleaning.rs` (tree_cleaning / prune_html / strip_tags).

**Confidence:** UNCHANGED MEDIUM. Fix locus is at parser-integration (html5ever
behaviour for `<td>`/`<tr>` outside `<table>` context), NOT a one-liner.

### 5d. Hypothesis F (NEW — curly-quote preservation, 5 records / 2 unique shas)

**Manifest.** Source HTML contains plain ASCII apostrophes/quotes (`'thing'`,
`'this doesn't include'`). Python's `trafilatura.extract` somewhere maps these
to curly quotes only on Python's output — empirically the reverse: oracle has
plain ASCII, mdrcel preserves the source's curly quote.

Wait — re-reading the windows: oracle has `the thing that the creator` (plain
`t`); mdrcel has `the ‘thing’ that the creator` (curly `'…'`). The oracle has
NO quotes there at all; mdrcel has curly quotes wrapping `thing`. This is not
a quote-style preservation issue, it's a **content-discard difference**:
Python discarded the `'` characters around `thing`; mdrcel kept them.

Inspecting the raw HTML for `sha=1d3fc86310f7`: the source likely has
`<i>'thing'</i>` or similar with `<i>` tags. Python's `tree_cleaning` strips
`<i>` (MANUALLY_STRIPPED). The curly quotes are TEXT content inside the
stripped element. Python's strip_tags merges the text into the parent's text
run; mdrcel may be doing the same; but somewhere on Python's path the
quotes-with-no-content-between-them collapse differently.

**Not a clean signature.** 5 records / 2 unique shas is too small to
hypothesise a fix locus.

**Confidence:** LOW. Deferred.
