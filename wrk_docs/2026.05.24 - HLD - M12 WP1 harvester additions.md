## 1. Goal

Three additions to `benchmark/fuzz/harvest.py` enable the 50K broad-sweep
corpus harvest. `--append` allows incremental accumulation across multiple WARC
segments without wiping prior work. `--max-per-domain N` caps pages per
registered domain to ensure corpus diversity. `--loose-filter` relaxes the
content filter to admit structurally unusual pages for the adversarial subset.
All three are additive; existing non-append, non-loose behaviour is unchanged.

## 2. Changes

### 2.1 `--append` mode

**What changes.** When `--append` is passed, the idempotent wipe at lines
217-219 of `harvest()` is skipped. The manifest is opened for append instead
of truncation. Before streaming begins, existing state is loaded:

- Parse the existing manifest JSONL and collect every `"sha"` value into
  `seen_sha` (currently initialised empty at line 226).
- Count existing entries so `kept` starts at the current total (currently
  initialised to 0 at line 224), enabling correct `--target-count` interaction:
  harvesting stops when cumulative `kept >= target_count`.

When `--append` is NOT passed, lines 217-219 execute as before (wipe + recreate).

**Where.**

- `harvest()` signature (line 207): add parameter `append: bool = False`.
- Lines 217-219: wrap the `shutil.rmtree` / `os.makedirs` block in
  `if not append:`.
- Lines 224-227: if `append` and manifest exists, pre-populate `seen_sha` and
  set `kept` to the count of loaded entries.
- Lines 311-313 (manifest write): change from truncating write (`"w"` mode) to
  append (`"a"` mode) when `append` is True. Only the new entries are written.

**Pseudocode for pre-population:**

```python
if append and os.path.isfile(manifest_path):
    with open(manifest_path, "r", encoding="utf-8") as fh:
        for line in fh:
            line = line.strip()
            if not line:
                continue
            rec = json.loads(line)
            seen_sha.add(rec["sha"])
    kept = len(seen_sha)
```

The `manifest_lines` list collects only NEW entries. At the end, it is appended
(not written) to the manifest file:

```python
if append:
    mode = "a"
else:
    mode = "w"
with open(manifest_path, mode, encoding="utf-8", newline="\n") as fh:
    if manifest_lines:
        fh.write("\n".join(manifest_lines) + "\n")
```

### 2.2 `--max-per-domain N`

**What changes.** A `collections.Counter` tracks how many pages have been kept
per registered domain. Pages are skipped once their domain reaches the cap.

**Domain extraction heuristic.** Parse the hostname from `WARC-Target-URI`
(using `urllib.parse.urlparse`), split on `.`, and take the last two labels.
Exception: if the second-level label is in a hardcoded set of country-code
SLDs, take the last three labels instead.

```python
from urllib.parse import urlparse

_CC_SLDS = frozenset({
    "co.uk", "com.au", "co.jp", "co.kr", "co.nz", "co.za",
    "com.br", "com.cn", "com.mx", "co.in",
})

def _registered_domain(url: str) -> str:
    """Extract registered domain from URL (diversity heuristic, not DNS)."""
    try:
        host = urlparse(url).hostname or ""
    except Exception:
        return ""
    parts = host.lower().split(".")
    if len(parts) >= 3:
        sld = ".".join(parts[-2:])  # e.g. "co.uk"
        if sld in _CC_SLDS:
            return ".".join(parts[-3:])
    if len(parts) >= 2:
        return ".".join(parts[-2:])
    return host
```

**Where.**

- New module-level constant `_CC_SLDS` and function `_registered_domain()`
  (placed near line 98, alongside the other helper regexes/functions).
- `harvest()` signature: add parameter `max_per_domain: int = 50`.
- Inside `harvest()`, after `seen_sha` init (line 226): add
  `domain_counts: Counter[str] = Counter()`.
- If `append` and pre-populating from manifest, also load domain counts:
  `domain_counts[_registered_domain(rec["source_url"])] += 1` for each entry.
- After the `sha in seen_sha` check (line 267-268): insert the domain check:

```python
if max_per_domain > 0:
    dom = _registered_domain(uri)
    if domain_counts[dom] >= max_per_domain:
        continue
    # (increment domain_counts[dom] after the page is confirmed kept,
    #  just before appending to manifest_lines)
```

- After line 293 (appending to `manifest_lines`): `domain_counts[dom] += 1`
  (only when `max_per_domain > 0`; when disabled, skip tracking entirely to
  avoid pointless urlparse calls).

### 2.3 `--loose-filter` flag

**What changes.** When set, the content filter skips the `<p>` tag /
visible-text-length check (lines 116-118 of `is_content()`). Only size bounds
(1 KB -- 2 MB, line 112) and control-char ratio (< 2%, line 114) apply.

**Implementation.** Add a `loose` parameter to `is_content()`:

```python
def is_content(raw: bytes, loose: bool = False) -> bool:
    n = len(raw)
    if n < 1024 or n > 2 * 1024 * 1024:
        return False
    if _control_char_ratio(raw) > 0.02:
        return False
    if not loose:
        n_p = len(P_RE.findall(raw))
        if n_p < 3 and _visible_text_len(raw) < 200:
            return False
    return True
```

**Where.**

- `is_content()` (line 105): add `loose: bool = False` parameter. Wrap lines
  116-118 in `if not loose:`.
- `harvest()` signature (line 207): add parameter `loose_filter: bool = False`.
- Line 255 call site: `if not is_content(raw, loose=loose_filter):`.
- The FROZEN filter docstring (line 106-109) gains a note: "When `loose=True`,
  the `<p>` tag / visible-text check is skipped (M12 adversarial subset only)."

**Invariant preserved.** When `loose=False` (the default), `is_content()`
behaves identically to today. The `_FILTER_FROZEN_AT` sentinel is not bumped
because the frozen filter itself is unchanged; `--loose-filter` is a separate
opt-in path.

## 4. Edge cases

**`--append` on first run (no existing manifest).**
The pre-population block checks `os.path.isfile(manifest_path)`. If the file
does not exist, `seen_sha` starts empty and `kept` starts at 0 -- identical
to non-append mode. The output directory is not wiped (it may not exist; we
still call `os.makedirs(output_dir, exist_ok=True)`).

**`--max-per-domain 0`.**
Domain tracking is disabled entirely. No `urlparse` calls, no counter, no
skipping. Equivalent to the current behaviour.

**`--loose-filter` without `--append`.**
Works correctly. The output dir is wiped, the manifest is truncated, and
harvesting starts fresh with the loose filter. This is an unusual combination
(the parent HLD recipe always pairs `--loose-filter` with `--append`) but it
is a valid workflow for standalone adversarial harvesting into a fresh corpus.

**`--target-count` interaction with `--append`.**
In append mode, `kept` is initialised to the number of existing manifest
entries. Harvesting stops when `kept >= target_count`. This means
`--target-count` specifies the cumulative total, not the number of new pages.
This is correct per the parent HLD section 3.3: "stop when the TOTAL manifest
reaches 52K entries." If the manifest already has >= target_count entries,
the harvester prints a summary and exits without streaming anything.

**Duplicate SHAs across WARC segments.**
Already handled: `seen_sha` is pre-populated from the manifest in append
mode, so the same page appearing in multiple WARCs is deduplicated.

**Corrupt manifest line.**
A malformed JSON line in the manifest during pre-population will raise a
`json.JSONDecodeError` and abort. This is the correct behaviour: a corrupt
manifest should not be silently appended to.

## 5. Test plan

Manual verification steps (no unit tests for this CLI script):

1. **Non-append baseline (regression).**
   Run with default args (`--target-count 10`, no `--append`). Verify output
   dir is wiped and recreated, manifest has exactly 10 entries, all SHAs on
   disk match their filenames.

2. **Append mode.**
   Run step 1. Then re-run with `--append --warc-index 1 --target-count 20`.
   Verify manifest now has 20 entries (10 original + 10 new), no files from
   the first run were deleted, no duplicate SHAs in the manifest.

3. **Append early exit.**
   Run step 2. Then re-run with `--append --warc-index 2 --target-count 15`.
   Verify the harvester exits immediately (manifest already has 20 >= 15),
   manifest is unchanged.

4. **Max-per-domain.**
   Run with `--max-per-domain 2 --target-count 50 --verbose`. Inspect
   manifest: no registered domain should appear more than 2 times. Verify
   with: `jq '.source_url' manifest.jsonl | sort | uniq -c | sort -rn | head`.

5. **Loose filter.**
   Run with `--loose-filter --target-count 20`. Compare kept count to a run
   without `--loose-filter` on the same WARC index. The loose run should
   accept more pages (or at minimum the same number).

6. **Loose + append (the recipe).**
   Run the full parent HLD recipe from section 3.3 at small scale
   (`--target-count 30` main, then `--target-count 35 --loose-filter --append`).
   Verify manifest accumulates correctly.

## 6. Open questions

1. **Domain counter in summary output.** The parent HLD does not mention
   reporting domain distribution in the harvest summary JSON (lines 315-321).
   Should we add a `"unique_domains"` count, or is that unnecessary noise?
   **Recommendation:** add it -- it is one line and useful for verifying
   diversity.

2. **Pre-existing files in output dir without manifest.** In append mode, if
   the output directory exists but the manifest does not, we start with an
   empty `seen_sha`. This could create a manifest that is missing entries for
   existing `.html` files on disk. This seems unlikely in practice (both are
   created together), but worth noting. **Recommendation:** accept this edge
   case; the operator is responsible for keeping the manifest and output dir
   in sync.
