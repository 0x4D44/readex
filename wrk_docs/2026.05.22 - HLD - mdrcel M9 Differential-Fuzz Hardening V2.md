## 1. The design in one paragraph

Our fidelity is proven on **51 hand-curated snapshots** — structurally "testing on the training set."
M9 widens that to **low-thousands of real-world pages from a Common Crawl slice** and runs the
existing **one-reference byte-diff** (mdrcel vs pinned Python `trafilatura`) at scale. But before
building any apparatus, a **Stage-0 spike** (§6.0) cheaply tests M9's own premise — *do thousands of
new pages reveal new root-cause clusters, or just re-hit the three we already know?* If the latter, we
stop. If it pays off, divergences funnel through a **mechanical-group-then-batch-LLM** triage into the
existing 3-bucket discipline; only **minimised reproducers of confirmed, legitimate-content bugs are
promoted** into the committed gate corpus. No new heavyweight deps, no feature flags, no env vars;
opt-in is `#[ignore]` + explicit invocation.

---

## 2. Goal & non-goals

**Goal.** Find port-fidelity bugs the curated corpus cannot surface, triage them efficiently, fix the
real ones, and lock each as a permanent regression — *only if* Stage 0 shows the bug yield is real.

**Non-goals (explicit):**
- **Not** a quality/benchmark exercise; **one-reference only** (mdrcel vs trafilatura). The three-way
  readability comparison answers a different question and belongs to a later track.
- **Not** metadata (that's M8 — §10 coordination).
- **Not** tens-of-thousands scale; **no** persistent oracle server, sharding, or distributed fetch.
- **Not** solving the known deferrals (blake2b fingerprint, U+2063); the pipeline *masks* them.
- **Not** testing charset detection or JS-rendering — both are explicitly the *caller's* job per the
  founding brief. Inputs are decoded to UTF-8 once and both sides see identical bytes (§5.2, §12).

---

## 3. Decisions locked

| Decision | Value | Rationale |
|---|---|---|
| Corpus source | Common Crawl slice | Realistic distribution for unknown-unknowns |
| First-run size | Low thousands | Stresses fidelity + triage without server-grade throughput |
| Reference count | One (mdrcel vs trafilatura) | Fidelity question |
| Input strategy | Real-world harvest; **no** pure-random; mutation only if Stage 0 justifies | Bug-density per input |
| Triage | Mechanical group → **ad-hoc batch LLM** → supervisor ratifies | Skill-ify only once stable |
| Minimiser | **Deferred** — build only if manual reduction proves too slow | No bug exists yet |
| Bulk corpus | Ephemeral; manifest-reproducible; git-ignored | Repo-weight; reproducible |
| Permanent artifacts | Minimised reproducers of *confirmed, legitimate-content* bugs | Fuzzing feeds the gate |
| **Milestone gate** | **Stage-0 spike go/no-go (§6.0)** | Falsify the premise cheaply first |

## 4. Pipeline & data flow

```
STAGE 0 (cheap, gates everything): few-hundred CC pages, existing per-doc oracle, masks OFF
        --> count distinct first-diff loci + garbage-rate --> GO / NO-GO (§6.0)

if GO:
[CC WARC slice] --harvest--> fuzz_corpus/<sha>.html      (run ONCE, config-locked oracle)
                              + manifest.jsonl (sha,url) --batch oracle--> oracle_cache.jsonl
                                                                            (+ integrity header)
   (fast, Rust, re-runnable without Python)                                      |
                                                                                 v
                          [diff (identical html+url both sides) + classify + group on read]
                                                                                 |
                                                              divergences.jsonl  (group = read-time
                                                                                  group-by, no extra file)
                                                                                 |
                                          [batch triage: N sampled members/group --> LLM hypothesis
                                           + vendored-source verify + draft bucket/ADR (§7)]
                                                                                 |
                            +------------------------+------------------------+--------------------+
                            v                        v                                             v
                       substantive              deferred (draft)                          allowlist (DRAFT —
                       (fix; fix must            rubber-stamp                               SUPERVISOR RATIFIES,
                        pass ALL group                                                      never auto-commit)
                        members; promote
                        only legit content)
```

The slow Python step runs **once**; all iterative Rust work reads the cache. (The minimiser, if ever
built, is the only thing that would re-invoke Python — and only on a handful of representatives, via
the *same* config-locked oracle path; see §11 deferred.)

## 5. Component design (maximal reuse, minimal new surface)

### 5.1 Shared comparison helpers — *prerequisite refactor* (corrected)
**Reality (verified against the gate files):** NFC normalisation (`.nfc()`) is duplicated across
*all* gates, and the coarse diff/report helpers (`collapse_ws`, `first_diff_index`, `window_around`,
oracle-spawn, escape) are near-verbatim duplicated between e.g. `trafilatura_markdown_gate.rs` and
`trafilatura_txt_gate.rs`. The fingerprint / U+2063 / `<teiHeader>` masking, however, lives
essentially **only in `trafilatura_tei_gate.rs`** and is *format-specific* — it is **not** applied by
the txt/json/markdown gates.

**Change:** extract into `tests/common/mod.rs` a `normalize(output: &str, format: Format) -> String`
that always applies NFC + the shared coarse helpers, and applies fingerprint/header masking **only
for the formats that emit them** (parameterised by `format`, not blanket). Plus the shared
oracle-spawn + diff-window reporting. Refactor all gates to `mod common;`.
**Invariant, mechanically checked:** capture each gate's current output hash; after the refactor the
hashes must be byte-identical (a recorded pre/post comparison, not a "still green" assertion).
`common` carries `#[allow(dead_code)]` since each test crate uses a different subset.

The fuzz harness lives in the same test crate, so it shares `common` directly — no cross-crate
plumbing, no new workspace member, no public-API widening.

### 5.2 Harvester — `benchmark/fuzz/harvest.py`
Downloads N Common Crawl WARC file(s) over HTTPS from `data.commoncrawl.org`, streams records with
`warcio`, keeps `text/html` decoded to UTF-8 that pass a **content filter** (see below), light-
stratifies toward content-bearing long-form, samples to target count, and writes:
- `benchmark/fuzz_corpus/<sha>.html` (git-ignored)
- appends `benchmark/fuzz/manifest.jsonl`: `{sha, source_url, warc_path, warc_offset, length}`
  (committed — reproduces the corpus without storing it; **`source_url` is load-bearing**, §5.3)

**Content filter (defines the test population — documented, because tuning it tunes bug yield):**
exclude pages that are not content articles — soft-404s, JS-stub/empty bodies (little/no `<p>` text),
truncated payloads, size outside ~1KB–2MB. The filter's job is to keep inputs the crate is *meant* to
handle; pages it rejects are out of mdrcel's scope per the founding brief, so excluding them removes
noise, not signal.

**Language note:** Python is a *deliberate* exception (CLAUDE.md "unless otherwise specified"):
`warcio` is Python-native/battle-tested and this is oracle-adjacent data-prep run rarely. Project
logic (diff/group) stays Rust. `warcio` pinned in the oracle venv.

**Selection-bias caveat (explicit):** CC is English-heavy and under-represents mdrcel's hard shapes
(SEC/regulator filings, table-heavy financial docs). **The 51 curated snapshots remain authoritative
for those shapes**; a green CC run is a claim about the broad web distribution, *not* about the hard
corpus. Light stratification mitigates but does not eliminate this.

### 5.3 Batch oracle — extend `benchmark/oracles/trafilatura/run.py`
Add `--batch <corpus_dir> <manifest>`: one process, one `trafilatura` import, loops over the manifest
and for each doc invokes the **single shared oracle function** (the same one the existing per-doc
modes and any future minimiser use) with **config locked** and the **per-doc `source_url` threaded in
as `url=`** — identical to what mdrcel is fed. It replicates the TEI `with_metadata=True` forcing for
the `xmltei` field. Emits one JSONL line per doc with **only the formats we actually diff** (start
with txt + markdown; widen if Stage 0 shows signal elsewhere).

**Cache integrity (CRITICAL fix):** the first line of `oracle_cache.jsonl` is a header
`{traf_version, cfg_sha, run_py_sha, manifest_sha}`. `fuzz_diff.rs` refuses to run if any differ from
the live environment — a venv bump or cfg edit must not silently turn stale-cache divergences into
phantom "bugs."

### 5.4 Diff + classify + group — `tests/fuzz_diff.rs` (one `#[ignore]`d harness)
Invoked explicitly: `cargo test --test fuzz_diff -- --ignored --nocapture`. For each manifest doc it
feeds mdrcel the **identical `(html, source_url)`** the oracle saw, normalises both via
`common::normalize`, byte-compares; on mismatch computes the simple signature (§6) and appends to
`benchmark/fuzz/divergences.jsonl`. **Grouping is a read-time group-by over that file** (no separate
`clusters.json`). Reads `oracle_cache.jsonl` (after the integrity check); never spawns Python.

### 5.5 Batch triage — ad-hoc first, skill later
The first triage batch is a supervisor-driven prompt over **N sampled members per group** (not just
one representative — guards against false-merges hiding a second cause) following §7. Promote to a
committed `.claude/skills/fuzz-triage/` skill **only once the workflow has stabilised** across a run
or two. (Avoids committing reusable infra before its shape is known.)

### 5.6 Promotion (guarded)
A divergence root-caused as a real mdrcel bug **and fixed** has its reproducer copied into
`benchmark/corpus/snapshots/` as a permanent fixture, picked up by the existing gate. Guards:
- the fix must turn **all** members of the group byte-equal (not just one reproducer);
- **supervisor confirms the reproducer is legitimate content** (not a soft-404/JS-stub/spam page) —
  matching Python on garbage is overfitting noise, not correctness, and must never become a fixture;
- minimised if a minimiser exists; otherwise the smallest natural reproducer.

## 6. Stages

### 6.0 Stage 0 — falsification spike (gates the whole milestone)
Harvest a *few hundred* CC pages (reuse the §5.2 filter, hand-run is fine), run the **existing
per-doc oracle** with masks **off**, and tabulate the **distinct first-diff loci / shape-classes**
plus the **garbage-rejection rate** of the filter.
**Go/No-Go:**
- If the divergence loci are essentially the **three known clusters** (U+2063/`Cf` chars, blake2b
  fingerprint, TEI header) → **NO-GO**: report to Arthur, do not build Stages 1-5. The known classes
  are category-level and fixed once, not by more corpus.
- If **new root-cause loci** appear at meaningful frequency → **GO**: proceed, using Stage-0 data to
  pick which formats to cache (§5.3) and whether mutation is worth adding.
This spike is ~½ day and can save weeks. It is the cheapest possible test of M9's premise.

### 6.1–6.5 (only on GO)
1. **Stage 1** — shared-helpers refactor (§5.1) with mechanical pre/post hash check. Land early (M8
   shared seam).
2. **Stage 2** — config-locked batch oracle + cache-integrity header + URL threading + `warcio` pin
   (§5.3). Validate on the existing 51.
3. **Stage 3** — harvester + content filter + manifest + light stratification (§5.2).
4. **Stage 4** — diff/classify/group harness (§5.4, §6).
5. **Stage 5** — ad-hoc batch triage + guarded fix/promote loop (§5.5, §5.6, §7), measured by the
   §8 fidelity KPI.

---

## 6'. The grouping signature (deliberately dumb v1)

Start with the **simplest key that groups**: `(format, shape_class, first-differing-node tag)` where
`shape_class` ∈ {whitespace, text-content, attribute, missing-element, extra-element, structural}.
This may over-merge; that is acceptable because triage feeds the LLM **several members per group**
(§5.5), so a hidden second cause surfaces in a sampled member, and intra-group diff-shape variance is
emitted as a split signal. Refine the key *only* if groups prove unusably coarse — never up front.

---

## 7. Triage contract — asymmetric trust + anti-inversion

Per group, triage **forms a root-cause hypothesis, then opens the vendored Python source
(`benchmark/oracles/trafilatura/.venv/Lib/site-packages/...`) and confirms it** before asserting.
Trust is asymmetric by blast radius:

| Bucket | Meaning | Autonomy |
|---|---|---|
| **substantive** | mdrcel wrong; fixable | Self-proving, **but** the fix must pass *all* group members and the reproducer must be confirmed legitimate content before promotion (§5.6). |
| **deferred** | mdrcel wrong; blocked on a dep we won't add | Drafts ADR `wrk_docs/m9-deferred/<hash>.md`; supervisor rubber-stamps. |
| **allowlist** | **the reference is wrong, mdrcel is right** | Extraordinary — where one would hide one's own bugs. Drafts with vendored-source proof in `wrk_docs/m9-allowlist/<hash>.md`; **supervisor MUST ratify; never auto-committed.** |

The substantive-lane guard (all-members + legitimate-content) is the V2 addition that stops the LLM
baking "match Python on a garbage page" into a permanent fixture.

---

## 8. Definition of done — a measurable fidelity KPI (not "diminishing returns")

Reserve a **frozen held-out harvest slice** that is *never used for triage*. The KPI is its
**byte-equal page rate** (per format), measured before Stage 5 and after each triage iteration.
- **Done** = the held-out KPI **plateaus** (successive iterations move it < ε), *not* "backlog
  drained" — because masking and coarse grouping can shrink the backlog without improving fidelity.
- A NO-GO at Stage 0, or a KPI that starts already-high and doesn't move, is a legitimate "M9 found
  little; the port was already faithful" outcome — report it; don't manufacture work.

## 11. Deferred (do NOT build until evidence demands it)

- **Minimiser (`tests/fuzz_min.rs`).** Only if manual reduction of real reproducers proves too slow.
  When built: it must route through the *same* config-locked oracle path as the batch run, validate
  its first interestingness call reproduces the cached divergence byte-for-byte before shrinking, and
  **keep each candidate `Dom` alive during its interestingness test** — the rcdom `Drop` quirk
  (documented in `Cargo.toml` Stage-2d/4d notes) drains descendants of dropped trees even while `Rc`
  handles are held; a tight clone/mutate/drop loop will hit it.
- **Committed triage skill (`.claude/skills/fuzz-triage/`).** Only once the ad-hoc workflow stabilises.
- **Structure-aware mutation.** Only if Stage 0 shows the real-world harvest under-yields.

## 12. Simplicity ledger — what we deliberately did NOT build

No feature flags, no env vars (opt-in is `#[ignore]` + explicit `--ignored`; data-prep is a separate
script). No persistent oracle server. No new workspace crate. No migration/rollback/back-compat. No
bulk corpus in git. No three-way comparison. No ML/embedding clustering. **No minimiser, no committed
skill, no signature taxonomy beyond the dumb key — all deferred behind evidence.** Honest core after
V2: *spike → (if it pays) harvest → run oracle once → diff against cache → eyeball groups → fix &
promote real bugs.* ~5 parts, holdable in one head.

## 13. Risks

- **Premise risk** (DA): mitigated by the Stage-0 go/no-go (§6.0).
- **Garbage-input overfitting** (DA): mitigated by the content filter (§5.2) + substantive-lane guard
  (§7) + legitimate-content promotion gate (§5.6).
- **Oracle drift** (methodology): mitigated by single config-locked oracle path + cache-integrity
  header + URL threading (§5.3).
- **False-merge hides a bug** (methodology): mitigated by N-members-per-group to triage + variance
  split signal (§5.5, §6').
- **CC selection bias** (methodology/DA): mitigated by keeping the 51 curated authoritative for hard
  shapes + light stratification + explicit scope statement (§5.2).
- **Unbounded effort** (DA): mitigated by the §8 KPI-plateau exit + Stage-0 gate.

## 14. Escalation to Arthur (above the review team's authority)

The devil's advocate's strongest challenge is strategic, not technical: **the crate has never been
integrated into the consumer or published — the project's original purpose — yet this is the seventh
fidelity milestone.** A ~1-day experiment (wire the existing crate into the consumer's `extract_content`
shim and run the known-bad URLs against the founding brief's *coarse word-count* acceptance bar) could
reveal that byte-fidelity beyond what we have is not the binding constraint at all — which would make
M9 lower priority than shipping. The review team cannot decide ship-vs-harden; **Arthur should.** If
the call is "harden," M9-V2 stands and Stage 0 protects it. If "ship first," M9 waits.
