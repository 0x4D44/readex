## 1. The design in one paragraph

Two small programs — `run.py` (Trafilatura) and `run.mjs` (Readability-JS) — take an absolute HTML file path, run one pinned, well-validated extractor with no runtime network, and print **exactly one JSON object**. They share no code; they share one committed artifact, `contract.schema.json`. "Frozen and conformant" is proven by documented one-liners in each adapter's README that run it on two synthetic fixtures, validate stdout against the schema, exercise the failure path, and re-run to confirm same-machine byte-stability. Everything below is detail; if it doesn't fit in your head, it's wrong.

```
   python …/run.py  <abs.html>  ─┐                          ┌─ node …/run.mjs <abs.html>
     (re-execs into its venv)    ├─ one JSON object on stdout ┤   (jsdom inert + stderr-only console)
                                 └──── contract.schema.json ──┘   ← the one shared artifact (shape only)
   downstream (sibling HLD, out of scope): benchmark/src/oracle.rs spawns these exact entrypoints
```

## 2. Scope, ownership, sequencing

**In scope:** `contract.schema.json`; `benchmark/oracles/trafilatura/`; `benchmark/oracles/readability-js/`; two synthetic fixtures; per-adapter READMEs; repo-root `.gitattributes`.
**Out of scope (owned elsewhere):** the `mdrcel` crate; the Rust runner/report; similarity metrics; **the word-count metric (the sibling harness owns the only one that is scored)**; the real ≥50-URL corpus, gold set, snapshot tool; any extraction algorithm; JS rendering, network fetching, encoding detection.

**Build sequencing (resolves the cross-team cold-start; no blocking sign-off ceremony):**
1. **First commit on the branch is `.gitattributes`** (repo root), *before any fixture, lockfile, or snapshot exists*. `core.autocrlf=true` on the dev box + no `.gitattributes` would silently CRLF-mangle committed bytes and git does **not** retroactively renormalize — so this must precede every byte-stable artifact, on either team.
2. The sibling HLD creates the Cargo workspace skeleton (it owns `benchmark/Cargo.toml`, `.gitignore`). This HLD then populates `benchmark/oracles/**`.
3. The two adapters are independent and proceed in parallel. The contract is already consistent with the sibling (header); if a genuine conflict arises, Arthur arbitrates and the schema file is amended once.

**Honoured constraints:** dev sandbox (no migration/rollback/back-compat); no feature flags; no environment variables (interpreters, paths, fixtures are conventions); simplicity; solve only the problem at hand; no premature future-proofing.

**Cross-machine reproducibility — explicit waiver (not a footnote):** the parent brief calls reproducibility "non-negotiable" on the implicit belief that pinning yields cross-machine identity. §3.5 shows that is false for Trafilatura's native `libxml2` path. **Conscious v1 decision:** document-and-lock with reproducibility scoped to a *single declared CI host*; the committed baseline is valid only on that host; cross-host drift is triaged, not asserted. True cross-machine reproducibility requires the deferred per-oracle container — that is the only mechanism that delivers the brief's non-negotiable, and it is consciously deferred for v1, not assumed away.

## 3. The frozen contract

### 3.1 Invocation

```
python <repo>/benchmark/oracles/trafilatura/run.py     <abs.html> [--base-url <URL>]
node   <repo>/benchmark/oracles/readability-js/run.mjs <abs.html> [--base-url <URL>]
```

* Interpreter **+** script, exactly as the sibling invokes them; launcher literals are `python` and `node`. The Trafilatura adapter makes bare `python` correct **by construction** (§4 re-exec), so the launcher discrepancy on the dev box (bare `python` ≠ the venv interpreter) is a non-issue, not a deferred risk.
* Path is **absolute**; the adapter MUST NOT `chdir` or resolve it relative to its own directory.
* Bytes are read **raw** and passed to the library unmodified; the library does whatever decoding it does — that is part of the pinned algorithm (honest framing, not "no decoding happens").
* **No network at runtime.** (Environment *bootstrap* — `pip install`/`npm ci` — does need network and is a one-time documented step, not the deterministic hot path.) `--base-url`, when present, is supplied as each tool's document/source URL where the pinned version supports it; its *effect differs per tool* (Trafilatura: extraction-time source URL; jsdom: DOM base for relative resolution) — accepted, because `canonical_url` is not scored.

### 3.2 Output object

stdout MUST be **exactly one** JSON object and nothing else; all logs/warnings/parser noise go to **stderr**. Keys in this exact order:

```
oracle          "trafilatura" | "readability-js"   (hardcoded literal)
oracle_version  string | null                       (null if the library could not be imported / version unreadable)
title           string | null
text            string                              (the primary comparison surface)
html            string | null                       (faithful-where-free; never scored)
word_count      integer | null                      (informational only; consumer recomputes and ignores this)
canonical_url   string | null                       (faithful-to-tool; never scored)
language        string | null                       (best-effort)
ok              boolean
error           string | null
```

Invariants (also in the schema where expressible): `ok == (error == null)`; on the failure path `ok == false` **and** exit ≠ 0 always co-occur (the adapter MUST NOT split them; the consumer treats either alone as failure).

### 3.3 Serialization (normative — the two adapters use different language serializers)

* **UTF-8, non-ASCII emitted raw:** Python MUST pass `ensure_ascii=False`; Node `JSON.stringify` already does. Cross-adapter byte-equality is **explicitly NOT required** — only per-adapter run-to-run self-stability.
* **`text` is made well-formed Unicode before serialization, by one pinned primitive per language** (this governs the comparison surface, so it cannot be left to the implementer): Python `text.encode("utf-8", "replace").decode("utf-8")`; Node `text.toWellFormed()` (ES2024, Node ≥ 20 — already pinned, §6). Both apply the Unicode-standard replacement of each maximal ill-formed subsequence (lone surrogates) by exactly one U+FFFD. This is a normal `ok:true` result, not an `error`.
* **Keys in the §3.2 order**, object built in that order; no `sort_keys`.
* No `NaN`/`Infinity` (the only number, `word_count`, is a finite integer or `null`).
* Write the complete object, then flush; do not interleave anything else on stdout. No size cap in v1. (This does not guarantee OS-level atomicity for very large `html`; a truncated stdout is a parse failure the consumer already records as `oracle_error` — sibling §5.)

### 3.4 The `ok` / `error` / exit tri-state (encodes the Bug E2 lesson)

| State | stdout | `ok` | `error` | exit |
|---|---|---|---|---|
| Ran, found content | full object | `true` | `null` | `0` |
| Ran, **found little/nothing** (Readability `.parse()`→`null`, Trafilatura→no content) | full object, `text:""` | `true` | `null` | `0` |
| **Adapter itself failed** (unreadable/missing file, venv truly absent, tool exception it can catch) | full object: `oracle` set, `oracle_version` set-or-`null`, `title/text/html/canonical_url/language/word_count` = `null` (or `text:""`), `error` set | `false` | message | non-zero |

* "Found little" is a valid result, never an error — the exact distinction Bug E2 collapsed.
* Catchable failures still emit the envelope (the consumer gets a structured `error`); imports/version reads happen **inside** the guarded block so an `ImportError` still yields it. An **uncatchable** failure (runtime absent, process OOM-killed, syntax error) is *no JSON + non-zero exit* — a distinct catastrophic state the consumer detects as "no parseable stdout". We do not pretend the adapter can catch what kills it before it runs.

### 3.5 Determinism (honest scope)

* **Same machine, same pinned environment: byte-identical stdout run-to-run.** This is the only property the README check (§7) verifies, and it is sufficient for the consumer's regression diffing on its single declared CI host.
* **Cross-machine: NOT guaranteed.** `pip freeze`/`package-lock.json` pin the Python/JS layers but **not** the native `libxml2`/`libxslt`/charset libraries `lxml` binds (set by the platform wheel/OS) nor host ICU. Trafilatura parsing can differ across machines at identical pins. Mitigation is the §2 single-host waiver; the only true fix is the deferred container.
* No clock/locale/random in emitted fields; no runtime network; Trafilatura dedup disabled (§4).

## 4. Trafilatura adapter — `run.py`

Rules (exact API calls pinned and verified against the pinned version at build):

* **Self-correct the interpreter first.** Before anything else, ensure execution under the committed venv interpreter: locate it relative to `__file__`; if not already running under it, re-execute the script under it. This makes the bare-`python` launcher correct by construction. Only if the committed venv is genuinely absent does it emit the §3.4 failure envelope. (Mechanism is verified-at-build to behave correctly on the Windows dev box; the HLD fixes the property, not the syscall.)
* **One structured call with metadata:** `bare_extraction(..., with_metadata=True)` then convert via the returned document's **`.as_dict()` method** (the `as_dict=` *parameter* is deprecated in 2.x). Plain `extract()` returns text only and is insufficient. Pin **Trafilatura 2.x** (the verified API surface).
* **Explicit committed config, never ambient;** it sets `deduplicate=False` — Trafilatura's module-level dedup LRU is stateful and non-deterministic; that is the whole reason (no in-process-future hypotheticals).
* **Field mapping:** `oracle="trafilatura"`. `oracle_version` via `importlib.metadata.version("trafilatura")`, read inside the guarded block; `null` if unreadable. `title`,`text` from the structured result as-is; emit `title` verbatim (do not coerce `""`→`null`). `html`: `null` in v1. `word_count`: a simple count over `text`, informational only. `canonical_url`: tool's reported canonical or `null`. `language`: tool-reported if free (no `py3langid` dependency — null-acceptable per scope), else `null`. Trafilatura/lxml logs → stderr.

## 5. Readability-JS adapter — `run.mjs`

Rules:

* **jsdom inert & offline:** do not enable `runScripts`; do not install a `resources` loader (both off by default — that *is* the no-network/no-JS/deterministic-parse guarantee; add nothing more).
* **stdout hygiene — concrete:** a bare `new VirtualConsole()` is silent (it does *not* auto-route to stderr; the *default* JSDOM console is what reaches stdout). Construct `new VirtualConsole()` and attach explicit, **non-throwing** listeners that write to `process.stderr` for **`error`, `warn`, `info`, `dir`, and `jsdomError`**. The `error` listener is mandatory: a Node `EventEmitter` with an unhandled `'error'` emission *throws*, which would turn a benign CSS parse error into the §3.4 catastrophic no-JSON state. stdout is exclusively the one JSON object.
* **`.parse()` → `null` is "found nothing", not an error** → `ok:true`, `text:""`, exit 0.
* **Field mapping:** `oracle="readability-js"`. `oracle_version` via `createRequire(import.meta.url)("@mozilla/readability/package.json").version`, inside the guarded block; on any failure `oracle_version=null` (schema allows it — no startup hazard). `title`,`text`=Readability's title/`textContent` verbatim. `html`=Readability's `.content` (free, so populated; accepted asymmetry vs Trafilatura, never scored). `word_count`: simple count over `text`, informational. `canonical_url`: `<link rel="canonical">` href, emitted only if absolute or a `--base-url` was given; with no base URL a relative canonical is `null` (do **not** resolve against `about:blank`). `language`: Readability's returned `lang` (do not re-read the DOM). `dir`/`byline`/`excerpt`/`siteName`/`publishedTime` deliberately ignored per brief scope. jsdom warnings → stderr.

## 6. Reproducibility & pinning

* **Trafilatura:** full transitive freeze (`requirements.txt`, exact `==`), committed, **including the fallback cascade** — the Barbaresi-2021 SOTA *is* Trafilatura + its fallbacks; pinning the package alone under-specifies the algorithm. Committed explicit config. README records exact Python version, OS, and `libxml2` version.
* **Readability-JS:** exact `@mozilla/readability` + `jsdom` in `package.json`; committed `package-lock.json`; committed exact Node pin (`.nvmrc` + `engines`) — jsdom behaviour is Node-version-sensitive, recording is not enough.
* Pins = latest stable Trafilatura **2.x** / `@mozilla/readability` / `jsdom` as of 2026-05-17, recorded in the READMEs. `oracle_version` is emitted at runtime so version skew shows in the consumer's (out-of-scope) baselines.
* **Container deferred** for v1 (document-and-lock floor per brief). It is the only true cross-machine guarantee (§2 waiver); slots in later with zero contract change.

## 7. Proving the contract holds (no bespoke test framework)

The frozen artifact is `benchmark/oracles/contract.schema.json`. Each adapter's README documents the copy-paste check (using a **dev-time** schema validator — `python -m jsonschema` / `npx ajv-cli` — **not** pinned into the adapter's runtime closure, so it does not widen the §3.5 risk surface):

1. run adapter on `article.html` → validate stdout against `../contract.schema.json`; assert `ok:true`, exit 0, **substantive `text`**, and that `<head>/<script>/<style>` text does **not** leak into `text`, and that stdout stays one clean object despite a deliberately malformed CSS rule in the fixture (exercises the §5 stderr-only console path);
2. run on `empty.html` → assert schema-valid, `ok:true`, `text:""`, exit 0 (the **Bug E2 "found nothing"** guard);
3. run on a **nonexistent path** → assert schema-valid failure envelope: `ok:false`, `error` set, exit ≠ 0 (the **Bug E2 "blew up"** guard — the catchable-failure arm, no fixture needed);
4. re-run step 1 and `diff` — assert byte-identical (same-machine determinism, §3.5).

Two synthetic, committed fixtures only (explicitly *not* the sibling's real corpus): `article.html` (clear synthetic article containing `<head>/<script>/<style>` chrome **and** a malformed CSS rule) and `empty.html` (head/script only, no body prose). `example.com` is deliberately **not** a fixture and **not** asserted non-empty — it is the canonical empty placeholder on which a correct Readability adapter legitimately returns empty.
