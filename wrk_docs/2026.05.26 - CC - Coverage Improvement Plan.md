# Coverage Improvement Plan — readex 0.19.x

**Date:** 2026-05-26
**Companion to:** `wrk_docs/2026.05.26 - CC - Coverage Baseline Report.md`
**Goal:** drive aggregate branch coverage from **72.03 %** to **≥ 98 %** without weakening parity gates.
**Budget:** ≤ 83 missed branches at the finish line (currently 1155).

## Guiding principles (the anti-Goodhart contract)

1. **Test contracts, not branches.** Every new test names the production contract it pins in an `// rationale:` line. If you cannot articulate the contract in one sentence, the test is wrong.
2. **No mocks; no tautologies.** Tests call real production functions on real (synthetic-but-shaped) inputs and assert real outputs. Forbidden patterns: `assert!(true)`, `let _ = f(x);`, `assert_eq!(x, x)`.
3. **No parity-gate regressions.** The seven format gates + the equivalence + htmldate parity gates + the xpath conformance harness are **read-only** for this push. Their fixtures and assertions don't change.
4. **No faithful-divergence churn.** If a Python source line is the oracle, the existing doc comment is the test's `// faithful-divergence:` anchor. Don't add tests that lock in unintentional drift.
5. **Negative shapes deserve names.** A test named `empty_input_returns_none` is fine; a test named `cov_branch_47` is not. Reviewers should be able to grep the test name and immediately see the contract.
6. **Granular commits.** Commit after every stage — small, atomic deliveries make rollback safe and let the agent multi-task. Title each commit `tests(<module>): cover <contract>`.

## Stage shape

Each stage:

1. Picks **one cluster** of cold-spot functions sharing a fix shape.
2. Writes faithful unit tests in the file's existing `mod tests {}` block.
3. Runs `cargo test --lib <module>::tests` to confirm green.
4. Runs full `cargo +nightly llvm-cov --branch ... --summary-only` to confirm:
   - **module** branch % rose by the expected delta, AND
   - **no other module's** branch % regressed (catch accidental coupling).
5. Commits with a one-line subject `tests(<module>): close <N> branches across <funcs>`.
6. Records the new top-level branch % in `wrk_docs/coverage_baseline/progress.log`.

Stages 1-10 below are pre-sized to roughly close the 1072-branch gap in chunks of 80-150 branches each. Stage 11 is the "long tail" mop-up. The plan is sequential so the agent can resume mid-stream after any interruption.

## Workflow per stage

Each stage runs as a self-contained agent task:

1. **Survey:** open the target file's existing `mod tests {}` block and read every test name to understand the contract vocabulary.
2. **Drill:** open the file's `wrk_docs/coverage_baseline/html/.../<file>.rs.html` and find the red lines.
3. **Write:** add `#[test]` blocks to `mod tests {}`. Name them after the contract. Add the `// rationale:` line.
4. **Run lib tests:** `cargo test --lib <module>::tests` — must be green.
5. **Run full coverage:** `cargo +nightly llvm-cov --branch --no-fail-fast --workspace --ignore-filename-regex 'benchmark|tests' --summary-only --tests -- --skip fuzz` — copy the per-module deltas into `progress.log`.
6. **Commit:** subject `tests(<module>): close <N> branches`, body summarising which contracts the new tests pin.

## Guardrails the agent applies before commit

For every PR diff:

- `git diff src/ | grep -E '^\+.*assert!\(true\)' → must be empty`.
- `git diff src/ | grep -E '^\+.*let _ =' → must be empty (every new statement must have a use)`.
- `cargo +nightly clippy --tests` warnings-only is OK; no errors.
- No edits to `tests/`, `benchmark/`, or any file outside the source module under test.
- No edits to `Cargo.toml`/`Cargo.lock` (no new dependencies).

## When the plan needs to break

If a stage proves harder than its budget (genuinely-unreachable defensive arms, faithful-divergence asymmetries with the Python oracle, dead code that should be removed), the agent **pauses and asks Arthur**. The plan never tunes its way out of a contract problem with `#[cfg(coverage_nightly)]` opt-outs.

Acceptable mitigations, in order of preference:

1. Add the missing test (default).
2. Refactor the dead arm out — only if the deletion has zero behavioural change (rare).
3. Add a `// llvm-cov:branch-not-reachable: <one-sentence-rationale>` annotation and call out the residual in the final report.

`#[coverage(off)]` and `#[cfg_attr(coverage_nightly, coverage(off))]` are forbidden — they suppress the metric without solving the underlying contract gap.
