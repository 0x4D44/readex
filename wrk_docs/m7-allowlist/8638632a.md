# M7 XML Allowlist ADR — `8638632aa27b2f45.html` (single empty infobox cell)

**Gate:** `tests/trafilatura_xml_gate.rs` (`--xml` / `output_format="xml"`).
**Bucket pre-allowlist:** `content-mismatch` (Rust = 78,856 chars, Python =
78,849 chars; first-diff at byte ~352).
**Verdict:** anti-inversion-clean — single empty table-cell drift from
html5ever-vs-lxml table construction; format-independent.

## Diagnosis

A Wikipedia article (vector skin) with a company infobox. mdrcel emits exactly
ONE extra empty cell that Python omits:

```
rust:   …<cell>Founded</cell><cell/></row>…
python: …<cell>Founded</cell></row>…
```

Whole-document empty-cell count: mdrcel = 1, Python = 0 (verified). The 7-char
delta in a 78 KB document (0.009%) is this single `<cell/>`.

This is the identical mechanism already allowlisted for the DFIN/XBRL filing at
`wrk_docs/m7-allowlist/683d5643.md` ("single empty-cell drift"): html5ever and
lxml retain a structurally-empty infobox `<td>` (here, one whose only content
was an element stripped during cleaning) slightly differently, so the table-walk
sees one more empty cell on the mdrcel side. The body text is byte-identical —
the txt, json, csv, and markdown gates are all GREEN on this fixture — because an
empty cell contributes nothing to the flattened text; it only surfaces as XML
tree structure.

Trafilatura's empty-element removal is a fixed TWO-pass sequence (core.py:48-59
falsy pre-prune, then xml.py:94-103 `remove_empty_elements`), both walking in
document order (parent-before-child). When the trees differ by one empty leaf,
the two passes cannot converge to identical output without a parser-level match.

## Anti-inversion check

Closing this would require either re-implementing lxml's table construction in
html5ever (faithfulness-violating) or adding a fixpoint/extra empty-cell trim
beyond Trafilatura's exact two-pass semantics (risks over-removing real empty
cells and diverging from Python elsewhere). Neither is a clean win; allowlisting
documents the trade-off. Cross-reference: `wrk_docs/m7-allowlist/683d5643.md`.
