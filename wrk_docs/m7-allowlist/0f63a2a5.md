# M7 TXT Allowlist ADR — `0f63a2a5a5620b74.html` (Hacker News front page, Python over-extracts nav chrome)

**Gate:** `tests/trafilatura_txt_gate.rs` (`--txt` / `output_format="txt"`).
**Bucket pre-allowlist:** `content-mismatch` (Rust = 4,076 chars, Python = 3,861 chars; first-diff at byte 0).
**Verdict:** anti-inversion-clean — identical root cause to the markdown gate, format-independent.

## Why this is the same divergence as markdown

On the txt path Python opens its output with HN's site-nav block
(`Hacker News\nnew\n|\npast\n|\ncomments\n…`) and renders the story list flat
(one cell per line). mdrcel emits the story listing as a structured table
(`| 1. | | Zerostack …`) and omits the `<td class="pagetop">` nav chrome.

This divergence is rooted in **content selection** (the nav `<td>` survives the
cascade's pruning in Python) and **table-walk shape** — both of which run
identically on the markdown and txt paths. The `output_format` string only
governs the final serialization of the already-selected body, so the txt
output diverges with the same character (first-diff at byte 0) as markdown.

## Cross-reference

Full diagnosis (BODY_XPATH match, `prune_unwanted_sections` not keying on

## Anti-inversion check

Matching Python would mean deliberately including HN's nav bar and abandoning
the structured table render to mirror Python's flat-cell + included-chrome
output — inversion. Allowlisting is correct.
