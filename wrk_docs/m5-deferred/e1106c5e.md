# DEFERRED: `e1106c5e26712078.html` (PBS news article)

## Status

Deferred to **M6** (or later). Listed in
`tests/trafilatura_markdown_gate.rs::DEFERRED_KNOWN_DEFECT`.

This is **not** an allowlist entry — by anti-inversion doctrine, Python's
output is the faithful body and mdrcel diverges from it. The divergence
shape is awkward because Python extracts what looks like a sidebar
("Latest Stories") and Rust extracts what looks like the article body,
but the gate's job is byte-equivalence to Python, so mdrcel is the
buggy side by gate definition.

## Why mdrcel is wrong

Content-selection divergence in the `BODY_XPATH` cascade. Python's body
selector lands on a `<div>` chain that contains the page's "Latest
Stories" sidebar list — that becomes Python's substantive output. Rust's
body selector skips the sidebar chain and lands on a different `<div>`
chain that contains the actual article body — that becomes Rust's
output.

This is the **inverse** of the Stage 6g Verge byline tightening
(commit `bf9a60c`, where the same root-cause class — ptest XPath
absolute-vs-relative semantics — caused Rust to over-include a sidebar
div). On the Verge fixture, mdrcel was over-extracting and Python was
right; on the PBS fixture, **Python is over-extracting** (sidebar) and
mdrcel is right by content-judgement standards. But the gate enforces
faithfulness to Python, not content-judgement, so mdrcel is the buggy
side.

## What deferred work is needed

Trace which `BODY_XPATH` expression in
`SETTINGS_CONSTANTS::BODY_XPATH` matches the PBS DOM in each engine,
and identify which div Python's selector lands on vs Rust's. Likely
candidates:

1. **XPath expression-order semantics.** `BODY_XPATH` is an ordered
   tuple; the first matching expression wins. If Python's matcher
   returns an earlier-in-tuple match (sidebar chain) where Rust's
   matcher rejects that expression and falls through to a later one
   (article chain), the fix is to align Rust's per-expression
   match/no-match decision with lxml's.
2. **Predicate evaluation differences.** `BODY_XPATH` expressions use
   `contains(@class, ...)`, `starts-with(@class, ...)`, and
   union/intersection patterns. If mdrcel's XPath evaluator handles
   any of these differently from lxml on the PBS DOM (e.g. class-
   attribute tokenisation), the matcher flips.
3. **Subtree shape post-cleaning.** The PBS DOM enters
   `extract_content` after `tree_cleaning`/`prune_html`/`convert_tags`.
   If mdrcel's cleaning pipeline mutates the DOM into a shape where
   Python's selector no longer matches, the divergence is upstream of
   `BODY_XPATH` evaluation itself.

Suggested investigative recipe (M6 Stage X):

1. Add a one-shot probe in `extract_content` (under a debug
   `#[cfg(test)]` gate) that logs the matched `BODY_XPATH` expression
   index for each fixture. Run on PBS in both Rust and Python (via the
   Python oracle's `--debug` mode), compare the indices.
2. If they differ, traverse the cleaned DOM at each expression's
   evaluation point and identify the first DOM divergence.
3. Apply the smallest fix that converges body-selector behaviour. May
   require either a cleaning-pipeline alignment OR an XPath evaluator
   fix.

The Verge tightening from Stage 6g (parent-walk-to-document for the
`//p//text()` ptest) is a precedent: a single XPath-semantic fix in
one place unlocked two fixtures simultaneously. The PBS fix may
similarly generalise to other content-selection edge cases.

Suggested milestone: **M6**.

## What this ADR deliberately does NOT claim

This ADR does NOT claim Python's "Latest Stories" extraction is
content-correct (it arguably is not — the article body would be a more
useful extraction). It claims that BY THE GATE'S DEFINITION (faithfulness
to `trafilatura==2.0.0` markdown output), mdrcel's PBS output is the
diverging side and the M6 fix-target is to align mdrcel's body
selector with Python's. If a future scoping decision changes the gate
contract from "faithful to Python" to "content-judgement-optimal",
this ADR's verdict reverses and mdrcel becomes the right side. That
contract change is OUT of M5's scope.
