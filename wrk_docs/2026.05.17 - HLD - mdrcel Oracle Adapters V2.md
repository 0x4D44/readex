## 1. The whole design in one paragraph

Two small programs — `run.py` (Trafilatura) and `run.mjs` (Readability-JS) — each take an absolute HTML file path, run one pinned, well-validated extractor with no network, and print **exactly one JSON object** describing what it found. They share no code; they share one artifact: a committed JSON Schema (`contract.schema.json`). "The contract is frozen and the adapter conforms" is proven not by a bespoke Rust harness but by a **one-command self-test inside each adapter's own toolchain** that runs the adapter on two synthetic fixtures, validates stdout against the committed schema, checks the `ok`/exit semantics, and re-runs to confirm same-machine byte-stability. Everything below is detail; if it doesn't fit in your head, it's wrong.

```
                     benchmark/oracles/contract.schema.json   (the one shared artifact — data, not code)
                                       ▲ validates                       ▲ validates
   python …/run.py  <abs html> ───────┘                                  └─────── node …/run.mjs <abs html>
        (pinned venv)                  one JSON object on stdout                         (pinned Node + locks)
                │  self-test: run fixtures, schema-validate, re-run & diff  │
                └──────────────────────────────────────────────────────────┘
                                       │
        (out of scope, sibling HLD)    ▼  benchmark/src/oracle.rs spawns these exact entrypoints — the ONE Rust consumer
```

## 3. The frozen contract

### 3.1 Invocation (verbatim, reconciled with sibling §5)

```
python <repo>/benchmark/oracles/trafilatura/run.py     <abs-snapshot.html> [--base-url <URL>]
node   <repo>/benchmark/oracles/readability-js/run.mjs <abs-snapshot.html> [--base-url <URL>]
```

* Interpreter **+** script path, exactly as the sibling consumer invokes them. The launcher names are the literal strings `python` and `node` — **reconciliation item §9** (on this dev box bare `python` ≠ `python3` ≠ `py`; the Trafilatura adapter defends against the wrong interpreter — §4).
* The path argument is **absolute**; the adapter MUST NOT `chdir` or resolve it against its own directory.
* The adapter reads the file as **raw bytes** and passes them to the underlying library unmodified. It performs no transcoding of its own; the library does whatever decoding it does — that is part of the pinned algorithm (the honest framing, not "no decoding happens").
* **No network at runtime, ever.** (Environment *bootstrap* is a different thing — §6.) `--base-url`, when present, is supplied as the tool's document/source URL where the pinned version supports it; exact relative-link behaviour is whatever that version does (verified at build, not asserted).

### 3.2 Output object (the schema is authoritative)

stdout MUST be **exactly one** JSON object and nothing else; all logs/warnings/parser noise go to **stderr**. Keys emitted in this exact order:

```
oracle          "trafilatura" | "readability-js"   (hardcoded literal — always known)
oracle_version  string | null                       (null only if the library could not be imported)
title           string | null
text            string                              (the primary comparison surface)
html            string | null                       (faithful-where-free; NOT a comparison surface)
word_count      integer                             (informational only — see §4; consumer recomputes)
canonical_url   string | null                       (faithful-to-tool; NOT a comparison surface)
language        string | null                       (best-effort)
ok              boolean
error           string | null                       (invariant: ok == (error == null))
```

### 3.3 Serialization (normative — this is part of the contract)

The two adapters use different language serializers, so this is pinned explicitly:

* **UTF-8, non-ASCII emitted raw** — Python MUST use `ensure_ascii=False`; Node `JSON.stringify` already does. (Cross-adapter byte-equality is **explicitly NOT required** — only per-adapter run-to-run self-stability. State this so no one chases a non-requirement.)
* **Keys in the §3.2 order**, object built in that order; no `sort_keys`.
* **`text` is sanitised to valid Unicode before serialization:** unpaired surrogates / un-encodable scalars are replaced with U+FFFD, deterministically. This is a normal `ok:true` result, **not** an `error` — it prevents the consumer's `serde_json` from rejecting otherwise-valid extractions (a fresh Bug-E2-class trap if left unspecified).
* No `NaN`/`Infinity` (the only number, `word_count`, is a finite integer).
* The complete object is written in **one buffered write, then flushed**; no streaming/partial emission. No artificial size cap in v1 (don't predict the future); `html` may be large and is part of the single object.

### 3.4 The `ok` / `error` / exit tri-state (encodes the Bug E2 lesson)

| State | stdout | `ok` | `error` | exit |
|---|---|---|---|---|
| Ran, found content | full object | `true` | `null` | `0` |
| Ran, **found little/nothing** (incl. Readability `.parse()`→`null`, Trafilatura→no content) | full object, `text:""` | `true` | `null` | `0` |
| **Adapter itself failed** (unreadable file, wrong interpreter/venv, tool exception it can catch) | full object, `oracle` set, `oracle_version` set-or-`null`, `title/text/html/canonical_url/language` = `null`/`""`, `error` set | `false` | message | non-zero |

* "Found little" is a valid result, never an error — the exact distinction Bug E2 collapsed.
* On a **catchable** failure the envelope is still emitted (the consumer gets a structured `error`); imports are done **inside** the guarded block so an `ImportError` still yields the failure envelope. An **uncatchable** failure (interpreter/runtime absent, syntax error) is *no JSON + non-zero exit* — a distinct catastrophic state the consumer detects via "no parseable stdout". We do not pretend the adapter can catch what kills it before it runs.

### 3.5 Determinism (honest scope — do not overclaim)

* **Same machine, same pinned environment: byte-identical stdout run-to-run.** This is the only property the self-test (§7) verifies, and it is sufficient for the consumer's regression diffing on a fixed CI host.
* **Cross-machine: best-effort, NOT guaranteed.** `pip freeze`/`package-lock.json` pin the Python/JS layers but **not** the native `libxml2`/`libxslt`/charset libraries `lxml` binds (set by the platform wheel/OS), nor the host ICU. Trafilatura's HTML parsing can therefore differ across machines at *identical pins*. The Trafilatura README records OS + `libxml2` version; cross-machine baseline differences are **triaged, not asserted**. The only true cross-machine guarantee is the deferred container (§6) — stating this keeps that deferral honestly informed.
* No clock/locale/random in emitted fields; no network. Trafilatura's stateful dedup is disabled (§4).

## 4. Trafilatura adapter — `benchmark/oracles/trafilatura/run.py`

Read bytes → run Trafilatura once with explicit pinned config → map to the contract → print one object. Exact API calls are pinned and verified against the pinned version at build (a constraint, not a from-memory call sequence):

* **One structured call, metadata enabled.** The plain `extract()` wrapper returns text only and is **insufficient**; the adapter MUST use the single call form that returns structured metadata alongside text in one parse (`bare_extraction`-class, metadata enabled, dict result). Pin a **Trafilatura 2.x** release (the verified API surface; the calls target 2.x).
* **Explicit committed config, never ambient.** A committed Trafilatura configuration is passed explicitly; no user/working-dir config is inherited. The config **MUST keep `deduplicate=False`** (the default): Trafilatura's module-level dedup LRU is stateful across calls and would make output order-dependent; one-doc-per-process plus this pin guarantees determinism even if the seam is later driven in-process.
* **Interpreter/venv defence.** `requirements.txt` is a full transitive freeze of a clean venv; reproducibility holds only if the *interpreter* matches (lxml ships per-CPython-ABI wheels → different native libxml2 → different `text`). `run.py` asserts it is running the pinned Python (a recorded `python_requires` floor) and inside the committed venv; if not, it emits the §3.4 failure envelope with a clear `error` (catchable). The README documents the exact Python version and the one-command venv create + install.
* **Field mapping:** `oracle="trafilatura"` (literal). `oracle_version` via `importlib.metadata.version("trafilatura")`, read **inside** the guarded block; `null` if the library could not be imported. `title`,`text` from the structured result as-is. `html`: `null` in v1 (consumer ignores it; not a comparison surface; declared so no implementer agonises over "cheaply"). `word_count`: a simple non-authoritative count over `text` (informational; **the sibling harness recomputes its own and ignores this** — do not use the tool's internal counter, do not build cross-adapter parity machinery). `canonical_url`: the tool's reported canonical, else `null` (faithful-to-tool, not a comparison surface). `language`: tool-reported only if free (no `py3langid` dependency added — null-acceptable per scope, keeps the closure small/deterministic), else `null`. Trafilatura/lxml logs → stderr.

## 5. Readability-JS adapter — `benchmark/oracles/readability-js/run.mjs`

Read bytes → build an inert offline DOM with jsdom → run `@mozilla/readability` → map → print one object.

* **jsdom inert & offline (minimal correct option set).** Do **not** enable `runScripts`; do **not** install a `resources` loader (both off by default — that *is* the no-network, no-JS, deterministic-parse guarantee; do not cargo-cult extra options). `--base-url`, when present, is jsdom's document `url`; when absent, the default is used.
* **stdout hygiene — concrete mechanism.** jsdom's default console routing can reach stdout, and the legacy `VirtualConsole.sendTo` was renamed to `forwardTo` (jsdom 27+) and re-introduces default error routing. So: construct `new VirtualConsole()` and explicitly subscribe its console events **and** the `jsdomError` event to `process.stderr` only. stdout is exclusively the one JSON object. The self-test includes a fixture that provokes a jsdom CSS-parse error so this is actually checked.
* **`.parse()` → `null` is "found nothing", not an error** → `ok:true`, `text:""`, exit 0 (§3.4).
* **Field mapping:** `oracle="readability-js"` (literal). `oracle_version` via `createRequire(import.meta.url)('@mozilla/readability/package.json').version` (robust across Node 20–24; avoids the `assert`-vs-`with` import-attributes Node-version trap), read **inside** the guarded block; `null` on failure. `title`,`text`=Readability's title/`textContent`. `html`=Readability's `.content` (free, so populated — accepted asymmetry vs Trafilatura per parent brief; not a comparison surface). `word_count`: simple non-authoritative count over `text` (see §4 — consumer recomputes). `canonical_url`: `<link rel="canonical">` href; resolved only if absolute or a `--base-url` is given — with no base URL a relative canonical is `null` (do **not** resolve against `about:blank`). `language`: Readability's returned `lang` (already from `<html lang>`; do not re-read the DOM). Readability also returns `dir`/`byline`/`excerpt`/`siteName`/`publishedTime` — deliberately ignored per parent-brief scope. jsdom warnings → stderr.

## 6. Reproducibility & pinning

* **Trafilatura:** full transitive freeze (`requirements.txt`, exact `==`), committed, **including the fallback cascade** — the Barbaresi-2021 SOTA *is* Trafilatura + its fallbacks, so pinning the package alone under-specifies the algorithm. Committed explicit config. README records exact Python version, OS, and `libxml2` version (§3.5).
* **Readability-JS:** exact `@mozilla/readability` **and** `jsdom` pins in `package.json`; committed `package-lock.json`; committed exact Node pin (`.nvmrc`/`engines`) — jsdom behaviour is Node-version-sensitive, so recording is not enough, pin it.
* **`oracle_version`** is emitted at runtime so a version skew shows up in the consumer's committed baselines (the consumer owns baselines; out of scope here).
* **Bootstrap ≠ runtime.** "No network" is a **runtime** guarantee. Environment **bootstrap** (`pip install` / `npm ci`) requires network + toolchain and is a documented one-time step, not the deterministic hot path. Fully-offline reproduction is **out of scope for v1** (would need vendoring; deferred with containerisation).
* **Containerisation deferred (document-and-lock for v1).** Recommended-stretch per brief; the simplicity steer + dev sandbox make document-and-lock the v1 floor. A per-oracle `Dockerfile` slots in later with zero contract change — and is the *only* true cross-machine determinism guarantee (§3.5).

## 7. Proving the contract holds (no bespoke Rust)

The frozen artifact is `benchmark/oracles/contract.schema.json`. "Frozen + conformant" is checkable by a **one-command self-test per adapter, in that adapter's own toolchain** (Python: a tiny script using the pinned `jsonschema`; Node: a tiny script using a pinned validator). Each oracle directory stays fully self-contained; no new language; no second Rust subprocess mechanism; the `mdrcel` library's `cargo test` is untouched. The sibling's `oracle.rs` is the single Rust consumer and exercises these same entrypoints when that team builds — we do not duplicate it.

**Two synthetic, committed fixtures** (explicitly *not* the sibling's real corpus): licence-clean, stable, intent-explicit.

| Fixture | What the self-test asserts |
|---|---|
| `article.html` — a clear synthetic article that also contains `<head>`/`<script>`/`<style>` chrome **and** a deliberately malformed CSS rule | schema-valid; `ok:true`, exit 0; **substantive `text`**; head/script/style text does **not** leak into `text`; stdout stays one clean JSON object despite the provoked jsdom CSS error |
| `empty.html` — head/script only, no body prose | schema-valid; **the Bug E2 guard**: `ok:true`, `text:""`, exit 0 (Readability `.parse()`→`null` must NOT become an error) |

Plus, for both fixtures on both adapters: run twice, assert **byte-identical** stdout (same-machine determinism, §3.5). `example.com` is deliberately **not** a fixture and **not** asserted to yield non-empty text — it is the canonical *empty placeholder* page on which a correct Readability adapter legitimately returns empty; the parent brief's "non-empty text for a content page" intent is served by the synthetic `article.html`, and the real head/script-exclusion requirement is asserted there too.

## 9. Cross-team reconciliation — BLOCKING (the single hard dependency)

Both HLDs were written the same day by different sessions; the contract is **not frozen until these are signed off with the sibling Harness team**, recorded here:

1. **Invocation verbatim:** `python …/run.py <abs> [--base-url]` / `node …/run.mjs <abs> [--base-url]`; literal launcher names `python`/`node`; absolute path; adapter never `chdir`s. (Matches sibling §5.)
2. **`ok`/`error`/exit tri-state** per §3.4 (sibling §5 maps non-zero **or** `ok:false` → `oracle_error`; the exit code is the authoritative failure signal, the envelope is the structured reason).
3. **Word-count ownership:** the **harness owns the only word count** (sibling §5/§8). Adapters emit `word_count` as non-authoritative self-description; the sibling's definition (its own tokenizer, incl. lowercase+NFC) governs all comparison. This HLD builds **no** cross-adapter word-count machinery.
4. **`html` and `canonical_url` are faithful-to-tool, NOT comparison surfaces** — the consumer must not score on them.
5. **Serialization (§3.5)** — confirm the consumer's `serde_json` accepts the agreed form (raw UTF-8, U+FFFD-sanitised `text`, fixed key order).
6. **Schema ownership:** `benchmark/oracles/contract.schema.json` is produced by this team, consumed read-only by the harness; the sibling §5 field list is non-normative.
7. **Repo-root ownership:** this team adds only `.gitattributes`; the sibling owns the workspace `Cargo.toml` and `.gitignore`.
