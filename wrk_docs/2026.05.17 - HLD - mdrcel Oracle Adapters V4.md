## 1. The design in one paragraph

Two small programs — `run.py` (Trafilatura) and `run.mjs` (Readability-JS) — take an absolute HTML file path, run one pinned, well-validated extractor with no runtime network, and print **exactly one JSON object**. They share no code; they share one committed artifact, `contract.schema.json`. "Frozen and conformant" is proven by a committed dev-time self-test per adapter that runs it on two synthetic fixtures, validates stdout against the schema, exercises the failure path, and re-runs to confirm same-machine byte-stability. Everything below is detail; if it doesn't fit in your head, it's wrong.

```
   python …/run.py  <abs.html>  ─┐                          ┌─ node …/run.mjs <abs.html>
     (re-execs into its venv)    ├─ one JSON object on stdout ┤   (jsdom inert + stderr-only console)
                                 └──── contract.schema.json ──┘   ← the one shared artifact (shape only)
   downstream (sibling HLD, out of scope): benchmark/src/oracle.rs spawns these exact entrypoints
```

## 2. Scope, ownership, sequencing

**In scope:** `contract.schema.json`; `benchmark/oracles/trafilatura/`; `benchmark/oracles/readability-js/`; two synthetic fixtures; per-adapter READMEs + committed dev-time self-tests; repo-root `.gitattributes`.
**Out of scope (owned elsewhere):** the `mdrcel` crate; the Rust runner/report; similarity metrics; **the word-count metric (the sibling harness owns the only scored one)**; the real ≥50-URL corpus, gold set, snapshot tool; any extraction algorithm; JS rendering, network fetching, encoding detection.

**Build sequencing (resolves the cross-team cold-start; no blocking sign-off ceremony):**
1. **`.gitattributes` is committed — and `git add --renormalize . && git commit` run — before any byte-stable artifact (fixture, lockfile, snapshot, schema) is added, by whichever team commits first.** The corruption mechanism is precise: with `core.autocrlf=true` (the dev box) and no `.gitattributes` *in effect for that path at `git add` time*, git mangles bytes on check-in; adding `.gitattributes` later does **not** retroactively fix already-added index entries — only `--renormalize` does. (The repo already has history, so "first commit" is not literally possible; the invariant is "before the first byte-stable artifact", which none yet exist.)
2. The sibling HLD creates the Cargo workspace skeleton (it owns `benchmark/Cargo.toml`, `.gitignore`); this HLD then populates `benchmark/oracles/**`. The two adapters are independent and built in parallel.
3. The contract is consistent with the sibling §5 on entrypoints, the `ok`/exit tri-state, word-count ownership, and html/canonical-not-scored (cross-read 2026-05-17). **One interaction is NOT yet reconciled** and is flagged for Arthur/sibling: the sibling's 60 s subprocess timeout vs. a legitimately slow/large extraction (§3.3, §9). The schema file is the single shared artifact; the sibling's field-list is non-normative; Arthur arbitrates any genuine conflict and the schema is amended once.

**Honoured constraints:** dev sandbox (no migration/rollback/back-compat); no feature flags; environment variables minimized (one tightly-scoped internal re-exec sentinel is the sole exception — §4, justified there); simplicity; solve only the problem at hand; no premature future-proofing.

**Cross-machine reproducibility — flagged for Arthur, not self-granted.** The parent brief titles its reproducibility section "non-negotiable" yet also states container is "recommended, **not mandated** for v1 — document-and-lock is the floor." §3.5 shows pinning cannot deliver cross-machine byte-identity (native `libxml2`). These cannot all hold. **Controlling reading taken here (subject to Arthur's confirmation):** the brief's own "document-and-lock is the v1 floor" governs; cross-machine identity is a known v1 deviation discharged only by the deferred per-oracle container; the committed baseline is valid only on a single declared CI host; cross-host drift is triaged, not asserted. This is surfaced as an escalation, not a waiver an HLD can self-grant.

## 3. The frozen contract

### 3.1 Invocation

```
python <repo>/benchmark/oracles/trafilatura/run.py     <abs.html> [--base-url <URL>]
node   <repo>/benchmark/oracles/readability-js/run.mjs <abs.html> [--base-url <URL>]
```

* Interpreter **+** script, exactly as the sibling invokes them; launcher literals are `python` and `node`. The Trafilatura adapter makes bare `python` correct **by construction** (§4 re-exec).
* Path is **absolute**; the adapter MUST NOT `chdir` or resolve it relative to its own directory.
* Bytes are read **raw** and passed to the library unmodified; the library does whatever decoding it does — part of the pinned algorithm (honest framing, not "no decoding happens").
* **No network at runtime.** Environment *bootstrap* (`pip install` / `npm ci`) needs network and is a one-time documented step — see §4/§5 (the venv and `node_modules` are git-ignored and absent on a fresh clone; bootstrap is required before any run succeeds). `--base-url`, when present, is supplied as each tool's document/source URL where the pinned version supports it; its effect differs per tool (Trafilatura: extraction-time source URL; jsdom: DOM base for relative resolution) — accepted, because `canonical_url` is not scored.

### 3.2 Output object

stdout MUST be **exactly one** JSON object and nothing else; all logs/warnings/parser noise go to **stderr**. Keys in this exact order:

```
oracle          "trafilatura" | "readability-js"   (hardcoded literal)
oracle_version  string | null                       (null iff the library could not be imported / version unreadable)
title           string | null
text            string                              (NEVER null; the primary comparison surface; "" when none)
html            string | null                       (faithful-where-free; never scored)
word_count      integer | null                      (informational only; consumer recomputes and ignores this)
canonical_url   string | null                       (faithful-to-tool; never scored)
language        string | null                       (best-effort)
ok              boolean
error           string | null
```

Invariants: `ok == (error == null)`; on the failure path `ok == false` **and** exit ≠ 0 always co-occur (the adapter MUST NOT split them; the consumer treats either alone as failure).

### 3.3 Serialization (normative — the two adapters use different language serializers)

* **UTF-8, non-ASCII emitted raw:** Python MUST pass `ensure_ascii=False`; Node `JSON.stringify` already does. **Cross-adapter byte-equality is explicitly NOT required** — only per-adapter run-to-run self-stability.
* **`text` MUST be valid UTF-8 before serialization** (so the consumer's `serde_json` cannot reject an otherwise-valid extraction — a Bug-E2-class trap if unspecified). Each lone surrogate is replaced by U+FFFD using one pinned, never-raising primitive per language: **Python** `text.encode("utf-8","surrogatepass").decode("utf-8","replace")`; **Node** `text.toWellFormed()` (ES2024; the pinned Node ≥ 20, §6). The exact U+FFFD count/placement **may differ between the two adapters** on ill-formed input — this is accepted: it occurs only inside regions the two parsers already decode differently (so they are non-comparable noise regardless), and cross-adapter byte-equality is already disclaimed. The requirement is *validity*, not cross-adapter identity. This is a normal `ok:true` result, never an `error`.
* **Keys in the §3.2 order**, object built in that order; no `sort_keys`. No `NaN`/`Infinity`.
* The adapter builds the **complete** object in memory, then performs a **single write followed by flush**; it MUST NOT begin emitting until the object is complete (no partial/interleaved stdout). No size cap in v1. This guarantees no partial-emission; it does not guarantee OS-level atomicity for a very large `html` — a truncated stdout is a parse failure the consumer records as `oracle_error` (sibling §5), and the timeout interaction is the open item in §2/§9.

### 3.4 The `ok` / `error` / exit tri-state (encodes the Bug E2 lesson)

| State | stdout | `ok` | `error` | exit |
|---|---|---|---|---|
| Ran, found content | full object | `true` | `null` | `0` |
| Ran, **found little/nothing** (Readability `.parse()`→`null`, Trafilatura→no content) | full object, `text:""` | `true` | `null` | `0` |
| **Adapter itself failed** (unreadable/missing file, venv corrupt/unspawnable, tool exception it can catch) | full object, **fully field-determined**: `oracle` set, `oracle_version` set-or-`null`, `title=html=canonical_url=language=null`, `text=""`, `word_count=null`, `error` set | `false` | message | non-zero |

* "Found little" is a valid result, never an error — the exact distinction Bug E2 collapsed.
* Catchable failures still emit the envelope; imports/version reads happen **inside** the guarded block so an `ImportError` still yields it. The venv re-exec (§4) is wrapped so that a corrupt/unspawnable venv is a *catchable* failure that still emits this envelope; only a genuinely **absent** venv on a fresh, un-bootstrapped clone is the expected pre-bootstrap state (its `error` names the bootstrap command).
* An **uncatchable** failure (runtime absent, process OOM-/timeout-killed, syntax error) is *no JSON + non-zero exit* — a distinct catastrophic state the consumer detects as "no parseable stdout". We do not pretend the adapter can catch what kills it before it runs.

### 3.5 Determinism (honest scope)

* **Same machine, same pinned environment: byte-identical stdout run-to-run.** This is the only property the self-test (§7) verifies, and it is sufficient for the consumer's regression diffing on its single declared CI host.
* **Cross-machine: NOT guaranteed.** `pip freeze`/`package-lock.json` pin the Python/JS layers but **not** the native `libxml2`/`libxslt`/charset libraries `lxml` binds nor host ICU. Mitigation is the §2 single-host reading; the only true fix is the deferred container.
* No clock/locale/random in emitted fields; no runtime network; Trafilatura dedup disabled (§4).

## 4. Trafilatura adapter — `run.py`

Rules (exact API calls pinned and verified against the pinned version at build):

* **Self-correct the interpreter first, safely.** The committed `requirements.txt` only reproduces under the matching interpreter (lxml ships per-CPython-ABI wheels → different native libxml2 → different `text`). `run.py` resolves the venv interpreter relative to `__file__`, compares it to the running interpreter using **resolved real paths** (`realpath` + `normcase`, so symlink/short-path/case differences cannot cause a mismatch), and if they differ re-executes itself under the venv interpreter. Invariants: **re-exec happens at most once**, enforced by a single tightly-scoped internal sentinel (one internal env marker — the deliberate, justified exception to "no env vars": the minimal correct fork-bomb guard); the locate+re-exec step is itself wrapped so a *corrupt/unspawnable* venv emits the §3.4 failure envelope (it is not moved outside the guarded path). A **genuinely absent** venv (fresh clone — the venv is git-ignored, see §5/layout) yields the §3.4 failure envelope whose `error` names the one-time bootstrap command; this is expected pre-bootstrap, not a defect to triage.
* **One structured call with metadata:** `bare_extraction(..., with_metadata=True)` then convert via the returned document's **`.as_dict()` method** (the `as_dict=` *parameter* is deprecated in 2.x). Plain `extract()` returns text only and is insufficient. Pin **Trafilatura 2.x**.
* **Explicit committed config, never ambient;** it sets `deduplicate=False` — Trafilatura's module-level dedup LRU is stateful and non-deterministic.
* **Field mapping:** `oracle="trafilatura"`. `oracle_version` via `importlib.metadata.version("trafilatura")`, inside the guarded block; `null` if unreadable. `title`,`text` from the structured result as-is; `title` verbatim (do not coerce `""`→`null`). `text` then passed through the §3.3 well-formed primitive. `html`: `null` in v1. `word_count`: simple count over `text`, informational; `null` on the failure path. `canonical_url`: tool's reported canonical or `null`. `language`: tool-reported if free (no `py3langid` dependency — null-acceptable per scope), else `null`. Trafilatura/lxml logs → stderr.

## 5. Readability-JS adapter — `run.mjs`

Rules:

* **jsdom inert & offline:** do not enable `runScripts`; do not install a `resources` loader (both off by default — that *is* the no-network/no-JS/deterministic-parse guarantee; add nothing more).
* **stdout hygiene:** a bare `new jsdom.VirtualConsole()` is, in the pinned jsdom, already non-throwing on `error` (the constructor installs a no-op `error` listener) and forwards nowhere (silent — it does not reach stdout). Construct one and attach **only** a `jsdomError` → `process.stderr` listener (diagnostics). Do **not** call `forwardTo`/`sendTo` (that is what would route to stdout). stdout is exclusively the one JSON object; verified by the §7 malformed-CSS check.
* **`.parse()` → `null` is "found nothing", not an error** → `ok:true`, `text:""`, exit 0.
* **Field mapping:** `oracle="readability-js"`. `oracle_version` via `createRequire(import.meta.url)("@mozilla/readability/package.json").version` (the pinned 0.6.x ships no `exports` map, so the subpath resolves; §6 records this and a pin bump must re-verify it), inside the guarded block; `null` on any failure. `title`,`text`=Readability's title/`textContent` verbatim; `text` then passed through the §3.3 well-formed primitive (`toWellFormed()`). `html`=Readability's `.content` (free, populated; accepted asymmetry vs Trafilatura; never scored). `word_count`: simple count over `text`, informational; `null` on failure. `canonical_url`: `<link rel="canonical">` href, emitted only if absolute or a `--base-url` was given; with no base URL a relative canonical is `null` (do **not** resolve against `about:blank`). `language`: Readability's returned `lang` (do not re-read the DOM). `dir`/`byline`/`excerpt`/`siteName`/`publishedTime` deliberately ignored per brief scope. jsdom warnings → stderr.

## 6. Reproducibility & pinning

* **Trafilatura:** full transitive freeze (`requirements.txt`, exact `==`), committed, **including the fallback cascade** — the Barbaresi-2021 SOTA *is* Trafilatura + its fallbacks; pinning the package alone under-specifies the algorithm. Committed explicit config. README records exact Python version, OS, and `libxml2` version, and leads with the one-time bootstrap command.
* **Readability-JS:** exact `@mozilla/readability` + `jsdom` in `package.json`; committed `package-lock.json`; committed exact Node pin (`.nvmrc` + `engines`, Node ≥ 20 for `toWellFormed()`) — jsdom behaviour is Node-version-sensitive. README notes the `@mozilla/readability` pin is verified to ship no `exports` map (so the `oracle_version` subpath read resolves); any pin bump re-verifies this.
* Pins = latest stable Trafilatura **2.x** / `@mozilla/readability` / `jsdom` as of 2026-05-17, recorded in the READMEs. `oracle_version` emitted at runtime so version skew shows in the consumer's (out-of-scope) baselines.
* **Container deferred** for v1 (document-and-lock floor per brief). It is the only true cross-machine guarantee (§2/§3.5); slots in later with zero contract change.

## 7. Proving the contract holds (committed dev-time self-test, no runtime dependency)

The frozen artifact is `benchmark/oracles/contract.schema.json`. Each adapter ships a committed `selftest` (`selftest.py` / `selftest.mjs`) that uses a **dev-time-only** schema validator (`jsonschema` / `ajv`) declared as a *dev/optional* dependency — **not** in the adapter's pinned runtime closure, so it does not widen the §3.5 risk surface. Each self-test is one command (documented in the README) and asserts:

1. on `article.html` → schema-valid; `ok:true`, exit 0; **substantive `text`**; `<head>/<script>/<style>` text does **not** leak into `text`; stdout is one clean object despite a deliberately malformed CSS rule in the fixture (exercises §5 stderr-only console);
2. on `empty.html` → schema-valid; `ok:true`, `text:""`, exit 0 — the **Bug E2 "found nothing"** guard;
3. on a **nonexistent path** → schema-valid failure envelope, fully field-determined per §3.4: `ok:false`, `error` set, exit ≠ 0 — the **Bug E2 "blew up" (catchable)** guard;
4. on a tiny **lone-surrogate fixture** → schema-valid, `ok:true`, and `text` is valid UTF-8 (no raise; the §3.3 primitive works) — the guard for the round-3 defect;
5. re-run step 1 and `diff` — byte-identical (same-machine determinism, §3.5).

Two synthetic, committed fixtures only (explicitly *not* the sibling's real corpus): `article.html` (clear synthetic article with `<head>/<script>/<style>` chrome **and** a malformed CSS rule) and `empty.html` (head/script only, no body prose). Steps 3 and 4 need no fixture file (nonexistent path; an inline byte literal). `example.com` is deliberately **not** a fixture and **not** asserted non-empty — it is the canonical empty placeholder on which a correct Readability adapter legitimately returns empty.

## 8. Directory layout & ownership

```
.gitattributes                       # repo root — OWNED HERE; committed + `git add --renormalize` BEFORE any
                                     #   byte-stable artifact (§2.1). `* text=auto eol=lf`; fixtures/locks/schema `-text`.
benchmark/                           # workspace created by the SIBLING HLD (we depend on it; we add only oracles/)
└── oracles/                         # OWNED HERE, in full
    ├── contract.schema.json         # the shared artifact (shape/types/nullability)
    ├── fixtures/{article.html, empty.html}
    ├── trafilatura/{README.md, requirements.txt, trafilatura.cfg, run.py, selftest.py}
    └── readability-js/{README.md, package.json, package-lock.json, .nvmrc, run.mjs, selftest.mjs}
```

The venv (`trafilatura/.venv/`) and `readability-js/node_modules/` are git-ignored (sibling owns `.gitignore`) — **absent on a fresh clone**; the README bootstrap step creates them and must run before any oracle invocation or self-test. The sibling owns the workspace `Cargo.toml`/`.gitignore`; this HLD owns only the above. Neither team edits the other's tree.
