## 1. The design in one paragraph

A faithful Rust port of **Trafilatura v2.0.0** (Adrien Barbaresi, ACL-IJCNLP 2021 System Demos, descriptive only), pinned to the version the harness oracle adapter runs (`benchmark/oracles/trafilatura/requirements.txt:31`). **Algorithm spec source of truth is the Python source code at `trafilatura@v2.0.0`** — the paper covers approach + benchmark, not heuristic specifics. The port lands as a sibling module `src/trafilatura/*.rs` to M2's `src/readability/*.rs`, behind the same frozen `lib.rs` public surface. Selection between Readability (M2) and Trafilatura (M3) is via an additive `Options::extractor` enum (default = `Readability`). M2 is unchanged. Trafilatura is a **4-extractor cascade plus arbiter**: own XPath extractor → vendored readability-lxml fork → vendored jusText → baseline rescue; results compared by a 7-branch arbiter in `compare_extraction`. M2's score-side-table architecture does NOT transfer; M3 is XPath-first + handler dispatch + length-and-link-density filters + per-language stoplists. Anti-inversion carries from M2 with the explicit clarification that for M3 the gate is **gold-set Coverage/Precision + the consumer ONLY**, never aggregate corpus mean (because Trafilatura IS oracle #1, aggregate corpus mean reduces to Jaccard-vs-self, which is anti-inversion territory).

## 2. Fidelity anchor — what the harness actually scores

Same harness as M2: `benchmark/src/score.rs:267-279` resolves the reference as gold-if-present-else-Trafilatura. For M3 the gate is gold-set Coverage/Precision + the named acceptance (per §4.1 carried verbatim from M2) — *never* aggregate corpus mean (DA-M3 finding). Aggregate Jaccard-vs-Trafilatura is reported diagnostic; never gating.

### 2.1 The `Document.text` semantic basis — lxml `xmltotxt`, not naïve `textContent`

Trafilatura serializes its TEI-like internal tree via `xmltotxt(postbody, formatting)` (`core.py:349`). This is NOT equivalent to jsdom `Node.textContent` (M2's anchor). The serializer inserts paragraph separators between block-level converted tags (`<list>`, `<head>`, `<quote>`, `<cell>`, `<row>`, `<p>`). The Rust port's text emission must match `xmltotxt`'s exact whitespace behaviour. The Stage-0c BLOCKER gate (analogue of M2's parser-equivalence gate) is on the *converted tree* shape, NOT on `Node.textContent`.

### 2.2 No `_setNodeTag` fast/slow concern; `convert_tags()` IS the rewrite

`convert_tags()` rewrites HTML tags wholesale during `htmlprocessing.py:29-43`. The Rust port has no fast/slow split to worry about. **However (DA-B-4 finding):** `convert_tags` rewrites tag names on potentially thousands of elements per page; if the Stage 1b implementer reuses M2's `set_node_tag` slow-branch (allocate + reparent children), the cost is O(N) per element. **DECISION-F (folded into Stage 1b exit criteria):** validate `convert_tags` performance on the slowest corpus URLs (EDGAR 10-K class); if unacceptable, specify a faster in-place tag-rename primitive in `dom.rs`. Acceptable threshold: Stage 1b extraction time ≤ 2× the equivalent M2 extraction on the same URL.

### 5.1 dom.rs facade — **additive extension required** (DA-B-4)

§6 of M2 said "highest risk: dom.rs". For M3, dom.rs gets **additively extended** (not "reused unchanged" as the draft claimed). New ops Stage 0a must add:

- **`tail(elem) -> Option<String>`** — lxml-equivalent: text content of the next-sibling `Text` node, stopping at the next element/comment/PI sibling. Required by `link_density_test_tables`, `process_node`, and the `xmltotxt` formatter.
- **`delete_with_tail_preserve(elem)`** — delete `elem` and re-attach its `.tail` to the previous sibling (or parent if no previous sibling), matching lxml's `etree.strip_elements` semantics that `prune_html` relies on.
- **`document_order_triplets() -> Iterator<(NodeRef, Option<String>, Option<String>)>`** — yields `(element, .text, .tail)` triplets in document order. The primitive that drives `xmltotxt`'s formatting decisions. Required by Stage 2 handlers.

These additions are tested via the existing parser-equivalence gate **plus** new Stage-0a unit tests pinning each new op against a hand-built fixture. The parser-equivalence gate's byte-identity vs `149c686` may NOT regress.

### 5.2 Public API extension (DECISION-C, with M-1 clarification)

`Options::extractor: Extractor` (enum, `Default = Readability`). Additive; `#[derive(Default)]` on `Options` continues to work. `extract == extract_with(Options::default())` preserved (pinned by existing test).

`Extracted` extended with the following Trafilatura-emitted fields (DA-M-1 finding; all always-empty on the Readability path; cross-extractor invariant test pins this):

- `categories: Vec<String>` (default = empty)
- `tags: Vec<String>` (default = empty)
- `image: Option<String>` (default = None)
- `description: Option<String>` (default = None)
- `hostname: Option<String>` (default = None)
- `license: Option<String>` (default = None)

These extend `Extracted` additively. `#[derive(Default)]` carries; `..Extracted::default()` patterns continue to work. Version bump implication: 0.4.x → **0.5.0 MINOR** at M3 ratification (adding fields to a non-`#[non_exhaustive]` struct is technically breaking; pre-1.0 sandboxes a MINOR per Rust SemVer convention).

## 6. Stage 0 — Foundational work (BLOCKING all later stages)

Stage 0 is now **three sub-stages**, NOT the single "reuse unchanged" the draft claimed.

### 6.0 Stage 0a — `dom.rs` additive extension

Implement `tail`, `delete_with_tail_preserve`, `document_order_triplets` per §5.1. Re-run the M2 parser-equivalence BLOCKER gate after additions; gate file byte-identity vs `149c686` must hold. New ops unit-tested in `dom.rs`'s own test module.

### 6.1 Stage 0b — XPath engine + conformance table (DECISION-A ratified)

Greenfield XPath evaluator covering the operator surface DA-B-1 enumerated (~13-18 distinct constructs):

**Operator catalog (DA-revised; this is the contract):**
- **Axes:** `descendant-or-self` (`//`, `.//`), `child` (`/`), `self::` (`self::list`, `self::item`, `self::p`, etc. — TEI-name-aware), `attribute` (`@`).
- **Predicates:** `[N]` positional; `[predicate-list][N]` (predicate THEN positional); `[@attr="value"]`; `[@attr]`.
- **Functions:** `contains(string, string)` (with `@id|@class`-union-in-first-arg → string-value-of-first-node semantics — DA-B-1's load-bearing edge case); `starts-with(string, string)`; `translate(string, from, to)` (single-char translation tables only); `text()` (used in `external.py` arbiter).
- **Operators:** `or`, `and`, `|` (union), `=`.

**Conformance discipline (DA-M-6 ratified):** every supported operator/axis/function has rows in `tests/xpath_conformance.rs` of the form `(xpath_expr, input_html, expected_node_count, expected_first_node_id_or_text)` cross-checked against a Python `lxml` subprocess at test-build time. Test failure = build failure. The first conformance row for every `xpaths.py` pattern (≥17 patterns × ~3 representative inputs each ≈ 50+ conformance rows minimum).

**LOC budget (DA-revised):** ~2000-3000 LOC + conformance harness. This is the new realistic estimate; do not optimize toward the draft's bogus 500-1000.

### 6.2 Stage 0c — Trafilatura-equivalence BLOCKER gate

Tests/`trafilatura_equivalence_gate.rs`: parses ~10 representative URLs in BOTH Python (subprocess via the existing oracle adapter) AND Rust, captures the **converted tree** (post-`convert_tags()`) as a normalized XML serialization, asserts byte-identical or with a documented small whitespace allowance. Stage-0c BLOCKER analogue of M2's parser-equivalence gate. Required after Stage 1b lands (since `convert_tags` is Stage 1b's deliverable).

### 6.3 Stage 0 exit criteria

- Stage 0a: `dom.rs` additive ops landed + tested; M2 parser-equivalence gate still passes byte-identical to `149c686`.
- Stage 0b: XPath engine landed; conformance table passes all rows; XPath patterns from `xpaths.py:13-54` (BODY_XPATH) hand-spot-checked against Python `lxml` output on 5 corpus URLs.
- Stage 0c: Trafilatura-equivalence gate authored (skeleton only at Stage 0; Stage 1b activates it).
- `Options::extractor: Trafilatura` returns `Err(NotImplemented)` (Stage-0 floor honest).
- `Options::extractor: Readability` path unchanged (M2 invariants pinned).
- Doctrine green: clippy + fmt + 481+ tests.
