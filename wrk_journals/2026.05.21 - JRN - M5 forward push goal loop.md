## M5 stage plan

The shape of M5 is **iterative**: we don't know how many divergences exist until we run the diff once. So the stage plan is *adaptive*:

- **Stage 1 (scoping)** — Agent surveys Python `trafilatura.extract` default behaviour, designs the harness skeleton, returns a one-pager plan
- **Stage 2 (harness + first run)** — Build the Python expectations generator + Rust diff harness; run on all 51 snapshots; produce divergence triage
- **Stage 3 (triage report)** — Bucket divergences (whitespace, escape, ordering, semantic gap); decide which buckets to fix vs allowlist
- **Stages 4..N (fix loop)** — One agent per bucket, each landing a focused fix; re-run harness after each
- **Stage Final-1** — Promote the harness to a BLOCKER gate (`trafilatura_corpus_diff_gate.rs`); record final pass rate + allowlist
- **Stage Final** — Wrap-up commit + journal closure

Anti-inversion doctrine (carried from M3/M4): **Python source always wins**; agents flag any apparent Python-side bug for supervisor verdict rather than silently aligning Rust to a wrong-looking output.
