# M11 — Metric Pivot + Bounded Parser Pre-Processor

**Started:** 2026-05-24
**Lead:** Claude (team lead) + Arthur (supervisor)
**Predecessor:** M10 Post-Fuzz Fidelity Cleanup (closed 2026-05-24, commit `4f4d4be`)

## Why this milestone

M10 closed with held-out mean byte-parity at 84.80% (+12.73% vs pre-M10). Three §7
escalations (Phase 2F, 2G, 2H) converged on one root cause for the remaining
~15%: **html5ever (HTML5-spec-strict) vs lxml (permissive)** parser-layer
divergence, with **no equivalent filter in Python's pipeline**.

Strategy ratified with Arthur:

- **B before A.** Byte-equivalence-to-Python is the wrong success criterion for
  ~58% of the residue (selection-phase: both extractors picked valid main
  content, just different valid choices). A content-similarity oracle alongside
  byte-equality will reframe most of that pile from "failures" to
  "different-but-equivalent" and give us a principled metric to measure
  Phase A against.
- **Then A.** A bounded HTML pre-processor for the three known
  parser-permissiveness shapes (`</br>`, stray cells outside `<table>`,
  `<xmp>`). §7-deviating but justified empirically by Phase B's metric.

Variants rejected before kickoff (per discussion):

- Port libxml2 to Rust — non-starter (~150K LOC, decades of behaviour).
- FFI to libxml2 — would close most of 15% by construction but adds C build
  chain on Windows, requires DOM facade refactor, introduces memory-safety
  boundary. Held in reserve if A's coverage proves insufficient.
- Fork html5ever with permissive mode — held in reserve between A and FFI;
  same coverage as A at ~10x effort, only worth it if A turns into death by
  insertion-mode rules.

## Phase plan

1. **Phase B — Content-similarity oracle.** Add a similarity metric to the M9
   fuzz harness (`tests/fuzz_diff.rs`) alongside byte-equality. Output a
   second column on the held-out KPI table. Re-classify the existing 440
   working-slice divergences. Acceptance: reframed KPI table published; the
   similarity metric is reproducible, deterministic, and interpretable.
2. **Phase A — Parser pre-processor.** HLD-first; faithful spec citations
   for each of the three shapes; bounded patch in front of html5ever; KPI
   measured against BOTH byte-equality AND Phase B's similarity metric.

### 2026-05-24 — Phase B HLD v1 signed off

Design agent returned HLD v1. Empirical probe (`notes/m11-phaseb-probe/
compute_metrics.py`) computed 5 candidate metrics on 8 curated divergence
records (26 rows across formats). Findings:

- **Primary metric: character-level Sørensen-Dice on bigrams** (multiset).
  O(n), tracks normalized Levenshtein at r≈0.99 across all divergence
  classes. Correctly scores whitespace-only diffs as 0.99+, selection-phase
  as 0.82-0.98, empty-vs-nonempty as 0.0.
- **Secondary: token-set Jaccard.** Even cheaper; good "any overlap?" signal;
  rounds to 1.0 on whitespace-only (hence secondary).
- **Rejected:** Levenshtein (O(n*m), impractical at 50KB), lcsubstr_ratio
  (systematically underscores whitespace diffs — 0.57 where Dice gives 0.99),
  ROUGE-L (no discriminative advantage over Dice; more complex).

Three thresholds calibrated from probe data:
- ≥0.95: near-equivalent (whitespace-only, trivial structural diffs)
- 0.80–0.95: content-similar (different paragraphs but substantial overlap)
- <0.80: divergent (fundamentally different or broken)

All 26 probe rows match human intuition. No probe row is misclassified.

Seven open questions, all resolved per design-agent recommendations:
- Both overall + divergent means in summary (§7.1)
- Tag-stripped for xml (§7.2)
- Compute on held-out slice for KPI (§7.3)
- Hardcoded threshold constants (§7.4)
- HashMap for bigram counts (§7.5)
- Panic/error → 0.0 unconditionally (§7.6 — agent's own catch)
- Avoid clone, pass &str directly (§7.7 — agent's own catch)

HLD path: `wrk_docs/2026.05.24 - HLD - M11 Phase B similarity oracle.md`.
Implementation agent spawned.

### 2026-05-24 — Phase A planning begins

Phase A target: bounded HTML pre-processor for the three known parser-
permissiveness shapes from M10 Phase 2F/2G/2H escalations:

1. Stray `</br>` end-tags (Phase 2H: html5ever parses as `<br>`, lxml drops)
2. Stray `<td>/<tr>/<th>/<tbody>/<tfoot>` outside `<table>` (Phase 2F/2G:
   html5ever drops per HTML5 spec, lxml keeps)
3. `<xmp>` raw-text elements (Phase 2F: html5ever swallows content, lxml
   treats as normal element)

This is explicitly §7-deviating: mdrcel would add filtering that Python's
pipeline does NOT have. Justified by Phase B's metric — we measure
improvement against Dice similarity, not byte-mimicry.

Phase A HLD design agent being spawned.

### 2026-05-24 — Phase A HLD v1 signed off

Design agent returned HLD v1 covering all three parser-permissiveness shapes:

1. **Shape 1 (`</br>`):** Strip stray `</br>` end-tags entirely (lxml drops
   them; html5ever parses as `<br>` start tag per spec §13.2.6.4.7).
2. **Shape 2 (stray table cells):** Rewrite `<td>/<th>/<tr>/<tbody>/<tfoot>/
   <thead>` outside `<table>` nesting to `<div>`, preserving attributes. Table-
   depth counter tracks nesting; cells inside `<table>` are untouched.
3. **Shape 3 (`<xmp>`):** Rewrite `<xmp>`/`</xmp>` to `<div>`/`</div>` so
   html5ever parses inner content as elements rather than raw text.

Implementation: `fn preprocess_html(html: &str) -> Cow<'_, str>` inside
`Dom::parse()` at `src/readability/dom.rs:127`. Two-pass: quick-scan for
triggers (zero allocation if none found), then single O(n) forward transform.

Key design decisions:
- `Cow::Borrowed` fast path for the common case (no triggers)
- Table-depth counter with saturating-sub for malformed HTML
- "Followed by whitespace or `>`" guard to prevent prefix tag matching
  (`<thread>` must not match `<thead`)
- Attribute-value-aware scanning deferred (edge case accepted)
- `<noframes>` and `<listing>` excluded (no corpus evidence)
- Applies to both trafilatura and readability pipelines (shared `Dom::parse()`)

15 unit tests specified covering: no-op, all three shapes, case insensitivity,
nested tables, attribute preservation, prefix-tag guard, empty input, saturating
depth, multiple-shapes-together.

HLD path: `wrk_docs/2026.05.24 - HLD - M11 Phase A parser pre-processor.md`.
Implementation agent spawned.

### What landed

- **Phase B:** Content-similarity oracle alongside byte-equality. Two metrics
  (Dice bigrams primary, Jaccard secondary) computed inline on every divergent
  page. Three severity buckets (near-equivalent ≥0.95, content-similar 0.80-0.95,
  divergent <0.80). Reframed 84.8% byte-equivalence as 94.6% effective parity
  (truly divergent: 5.4%). Zero new dependencies; harness-only code.
- **Phase A:** HTML pre-processor for three parser-permissiveness shapes:
  (1) strip stray `</br>` end-tags, (2) rewrite stray `<td>/<th>/<tr>/<tbody>/
  <tfoot>/<thead>` outside `<table>` to `<div>`, (3) rewrite `<xmp>`/`</xmp>`
  to `<div>`/`</div>`. Integrated into `Dom::parse()` (single entry point;
  ~200+ call sites benefit automatically). `Cow::Borrowed` fast path for the
  common case. +1.00% mean byte-equality; zero regressions on 61 gate tests
  and 1230 lib tests.
