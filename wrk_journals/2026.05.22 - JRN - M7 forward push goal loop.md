## Charter

**M7 = output-format differential-gate expansion.**

Recon (Plan agent, 2026-05-22) reframed the premise. mdrcel **already implements** all six
trafilatura output formats — `extract_to_markdown / json / csv / xml / tei` in `src/lib.rs`,
serialized by `src/trafilatura/output.rs` (port of Python `xml.py`). txt is the
formatting-off variant of the markdown/`xmltotxt` path.

What is **missing** is corpus-wide BLOCKER **gates** for the non-markdown formats. Today only
`trafilatura_markdown_gate.rs` (markdown) and `trafilatura_extract_content_gate.rs`
(intermediate XML) exist. M7 adds gates for txt / json / csv / xml / xmltei, mirroring the
markdown gate's structure.

So each stage = (1) add a Python oracle mode + (2) clone the gate harness for that format +
(3) run the 51-fixture corpus byte-diff + (4) triage divergences using the established
three-bucket doctrine (substantive / `PYTHON_UNDER_EXTRACT_ALLOWLIST` / `DEFERRED_KNOWN_DEFECT`),
each non-substantive bucket entry backed by an ADR.

### Anti-inversion doctrine (carried from M3-M6 — earned, non-negotiable)
- Python source of truth is **vendored**: `benchmark/oracles/trafilatura/.venv/Lib/site-packages/trafilatura/*.py` (v2.0.0).
- ADRs are starting points, not recipes. Verify every root-cause claim against Python source.
- Allowlist = Python is wrong (mdrcel is right). Deferred = mdrcel is wrong (known defect).
  Fix cheaply when the fix is small and safe; otherwise bucket with an ADR and move on.
- Scope-reduce rather than block. If a format stalls, bucket the divergences and proceed.

### Reserve work (fill-in if a format stalls)
- rcdom shared-ownership audit (tier-1 systemic, 4 M5/M6 occurrences).
- jusText port hardening (generalise FRED's `custom_justext` fix as a differential test).

## Harness anatomy (from recon — what each gate must replicate)
- Corpus: 51 `benchmark/corpus/snapshots/*.html`, hardcoded `FIXTURES` array.
- Oracle: **live** `python run.py --<fmt> <path>` subprocess per fixture (no stored goldens).
- Compare: NFC-normalise both sides, strict byte equality.
- Buckets: untriaged enum + `PYTHON_UNDER_EXTRACT_ALLOWLIST` + `DEFERRED_KNOWN_DEFECT`;
  accounting invariant asserts pass+allowlist+deferred+buckets == total; GREEN when buckets empty.
- New oracle modes go in `benchmark/oracles/trafilatura/run.py` (mirror the `--markdown` branch).

## Known divergence risks flagged by recon
- **json**: `fingerprint`/`id`/`filedate` are `null` in mdrcel (output.rs:1216-1220) → likely allowlist.
- **csv**: header-row delta (mdrcel prepends a header Python's `xmltocsv` omits, lib.rs:830-834) + `\r\n`/quoting dialect.
- **xmltei**: TEI header/namespace/`check_tei` attribute whitelist; mdrcel explicitly does not validate (lib.rs:1064) — most format-specific surface.

### Supervisor decision (post-Stage 1) — U+2063 / `remove_control_characters`

Verified the dependency tree directly: only `unicode-ident`, `unicode-normalization`,
`unicode-xid` present — **none expose Unicode general category**. So a faithful
`remove_control_characters` (Python keeps `isprintable() or isspace()`, dropping non-space
C* categories: Cf, Cs, Co, Cn, and non-whitespace Cc) genuinely needs a new dependency
(`unicode-general-category` / `finl_unicode`) or a vendored category table. `char::is_control()`
only covers Cc.

**Ruling:** DO NOT add the dependency unilaterally. Rationale: (1) the crate keeps a deliberately
minimal 6-direct-dep footprint — a design value adding a dep would erode; (2) Arthur's process
treats dependency additions as decision records (DEC-3); (3) the deferred bucket makes this
**non-blocking** — gates still go GREEN carrying the deferred entry, so the loop is not stuck.
Deferring satisfies "make sensible decisions to avoid getting blocked".

**Plan:** Stages 2/4/5 (json/xml/xmltei flow the same body text through the same sanitize) will
likely re-surface U+2063 on fixture `507b9cdb`. They will add it to their deferred list
**referencing the existing `wrk_docs/m7-deferred/507b9cdb.md`** (no new ADRs — DRY). At M7 close
I'll present a single decision package: add `unicode-general-category`, implement
`remove_control_characters` faithfully in BOTH sites (`output.rs:787` + `metadata_url.rs:739`),
add a category-set differential test, and clear all U+2063 deferrals at once. Arthur decides
the dependency.

**Task A — metadata gating (the real Finding #1 fix).** The oracle calls
`extract(..., with_metadata=False)`; Python's `core.py:269-270` then builds an *empty*
`Document()`, so the 9 metadata-derived columns (url, id, fingerprint, hostname, title, image,

metadata unconditionally. Fix: added a `with_metadata` flag to `pub(crate) xmltocsv`
(`src/trafilatura/output.rs`) — when `false`, the metadata columns are forced to `null`,
mirroring the empty-`Document` semantics exactly like `build_json_output` already does.
`extract_to_csv` now passes `opts.with_metadata` (default `false`). The public `extract_to_csv`
signature is unchanged. Three internal `xmltocsv` test callers updated for the new parameter
(all use empty/default metadata, so no assertion changed; passed `true` where they exercise the
metadata-read path, `false` for the body-quoting test). `cargo test --lib` = 1156/1156, no
regression.

**Task B — fingerprint masking (Finding #2, corrected).** *Anti-inversion catch:* the brief
assumed mdrcel emits an FNV-1a hex fingerprint that "can never equal Python's". The actual csv
code emits `null` for the fingerprint column (no fingerprint is wired onto the csv `Document`;
M4 Stage 6 deferred). Python *does* emit a real value — `content_fingerprint(str(title) + " " +
raw_text)` runs for every non-txt format even with `with_metadata=False` (`core.py:481-485`),
via `Simhash(content).to_hex()` = `blake2b(digest_size=8)` (`deduplication.py:11,75,141-143`).
So the divergence is `null` (mdrcel, honest) vs lowercase-hex (Python). Either way the values
can never match by design (FNV-1a vs blake2b). Per the supervisor ruling — NO blake2b
dependency, NO change to `deduplication.rs` — the gate masks the single fingerprint column
test-side: a quote-aware reader parses both rows into 11 fields (the `text` cell embeds
`\t`/`\r`/`\n` and is csv-quoted, so `split('\t')` would be wrong), shape-checks Python's
field-2 as a well-formed lowercase-hex simhash (1–16 chars; `hex()` strips leading zeros, so
NOT hardcoded to 16), blanks field-2 on both sides, re-serialises, and byte-compares the
remaining 10 columns. A malformed-but-present Python fingerprint PANICS (the mask must never
hide a structural divergence). When a side is not a clean 11-field row (e.g. Python under-
extracts to empty — `41d2afac`), masking is skipped and the raw strings flow to the existing
allowlist triage. ADR: `wrk_docs/m7-deferred/fingerprint-blake2b.md`.

### Stage 4 — xml gate

**Result:** `tests/trafilatura_xml_gate.rs` GREEN — **39 substantive + 11 allowlisted + 1
deferred = 51/51**. Cloned the csv gate (subprocess `--xml`, `extract_to_xml`,
NFC + strict byte compare, three-bucket accounting + invariant).

**Three internal `output.rs` serialization fixes** (all behind unchanged public signatures —
this is why xml needed real fixes where csv/json did not: those flatten the tree to text, xml
serialises the tree shape itself):

1. **Sticky `formatting` flag in `serialize_xml_pretty`.** libxml2's `pretty_print` disables
   formatting for an element's ENTIRE descendant subtree once it hits mixed content (text or a
   child with a non-blank tail), and never re-enables it deeper. mdrcel re-decided per element,
   so a `<main>` made mixed by a `<head>`'s `[edit]` tail went flat but its `<table>` re-indented.
   Threaded a `formatting: bool` through `write_element_pretty`; once inline, descendants pass
   `false`. (Verified against lxml with isolated probes.)
2. **`sanitize_tree` (utils.py:315-336) now runs on the XML path** (xml.py:167 — was previously
   deferred; the old `serialize_xml_pretty` doc literally said so). Walks every element, runs
   `sanitize` over `.text`/`.tail` with `preserve_space`/`trailing_space` computed from
   `SPACING_PROTECTED`/`FORMATTING_PROTECTED`. Required adding those two knobs to
   `line_processing`/`sanitize` (default callsite `sanitize_text`→`xmltotxt` unchanged). This
   collapsed all 10 whitespace-only divergences (raw source `\n`/space runs inside element text)
   and the `<cell> Statutory…` leading-space-on-FORMATTING_PROTECTED cases.
3. **`prune_childless_textless` (core.py:48-59) added** as the missing FIRST empty-element pass.
   Python runs TWO document-order passes — the falsy pre-prune (`not text and not tail`) then
   `remove_empty_elements` (`text_chars_test`) — so an empty inner `<p>` is removed first and the
   now-childless `<cell>` cascades out in the second pass. A single parent-first pass cannot
   cascade. Removed ~4 fixtures' worth of stray `<cell/>`.

Net effect of the three fixes: 22 → 39 substantive.

### Reserve work — rcdom reparent-tail fix

(lxml `append`) and `dom::insert_with_tail(new_parent, child, idx)` (lxml `insert`). Both capture

orphan.
