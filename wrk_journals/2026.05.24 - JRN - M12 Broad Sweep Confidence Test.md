# M12 — Broad Sweep Confidence Test

**Started:** 2026-05-24
**Lead:** Claude (team lead) + Arthur (supervisor)
**Predecessor:** M11 Metric Pivot + Parser Pre-Processor (closed 2026-05-24, commit `a7ed9c5`)

## Why this milestone

M10/M11 brought byte-equivalence to 85.8% and effective parity (Dice ≥ 0.80) to
94.8% on the 1500-page M9 corpus. But the entire quality story rests on 1500
Common Crawl pages. Before mdrcel gains real consumers, we need confidence that
there aren't nasty corner cases lurking at scale — panics, content extraction
failures, or genuinely broken output on the diverse HTML the real web produces.

M12 is a one-off confidence sweep: run mdrcel and Python trafilatura on 50K+
diverse pages, compare every pair, surface surprises.

## Process discipline

- **HLD-first** for each work package; team lead reviews and iterates with
  design agents before signing off implementation.
- **Code-review discipline.** `/code-reviewer` and `/code-review-apply-feedback`
  skills on every implementation diff.
- **Commit policy.** Commits authorised only by team lead.
- **Journal cadence.** Updated after each significant step.

## Work packages

Two implementation packages (parallelisable):

1. **WP1 — Harvester additions** (`benchmark/fuzz/harvest.py`): `--append`,
   `--max-per-domain`, `--loose-filter` flags. ~100 lines Python.
2. **WP2 — Sweep harness** (`tests/fuzz_diff.rs`): `ExtractOutcome::Timeout`
   + 30s deadline in `safe_extract`, `ShapeClass::Timeout`, `broad_sweep()`
   test function. ~80 lines Rust.

Then three sequential pipeline steps (unattended):

3. **Harvest** — 50K+ pages from Common Crawl WARCs (~1-2 hours)
4. **Oracle** — Python trafilatura batch on all pages (~2-3 hours)
5. **Sweep** — `broad_sweep()` on full corpus (~30-60 min)

Finally:

6. **Triage** — analyse outlier report, surface findings.
