## Supervision model

- Per stage: **implement** (general-purpose agent, TDD) → **review** (`code-reviewer` skill) → **apply feedback** (`code-review-apply-feedback` skill) → **devil's advocate** challenge → fix loop → **supervisor commit**.
- Supervisor protects context: high-level direction only; agents return concise summaries (decisions, test pass/fail, files, blockers), never file dumps. No direct testing/debugging by supervisor.
- Sequential stages (each builds on the prior). Commit at every stage boundary (atomic, only authored files; user 0x4D44).
- Final: independent verification team compares implementation vs HLD; iterate until 100%.
