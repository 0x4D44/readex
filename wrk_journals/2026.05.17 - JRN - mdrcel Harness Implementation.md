- **R1 — Stage 4 blocked:** `benchmark/oracles/contract.schema.json` absent (oracle team not delivered). Scoring/report/regression proceed against fixture JSON. Flagged to oracle team.
- **R2 — Cargo.lock byte-stability — CORRECTED (devil's advocate R3, 2026-05-17).** My original framing here was wrong and seeded a false rationale in commit `8541ea2`'s message. **Forensic fact:** `bfeccf4:Cargo.lock` had **zero CR bytes** when first committed; `core.autocrlf=true` strips CR on check-in, so the blob was already LF-clean and `git add --renormalize .` was a **no-op** for it. The commit message's "repairs pre-committed Cargo.lock byte-stability" is **factually false** — the outcome (byte-stable) is correct, but not for the stated reason. The commit is **not** amended (CLAUDE.md). **Accurate rationale, recorded here as the authoritative record:** committing `.gitattributes` before any `-text` artifact is **prophylactic for *future* byte-stable artifacts** — the HTML snapshots (Stage 3) and the oracle `contract.schema.json` (Stage 4), which MUST have the `-text` rule in effect *at `git add` time* or `text=auto` will mangle them irreversibly. Cargo.lock was never at risk. The ordering discipline stands; only the Cargo.lock-specific justification was fiction. Future sessions/oracle team: do not cargo-cult "renormalize repaired the lock."
- **R3 — mdrcel is a binary (`src/main.rs`):** parent brief mandates a library crate. Bootstrap converts to lib; Stage 5 defines the real API surface.
- **D1 — Feature branch:** branched off `main` per CLAUDE.md (default-branch rule) + multi-agent concurrency (oracle team + `wrk_b2_spike/` active).
- **D2 — Scope hygiene:** commits are atomic and contain only files this team authored. `wrk_b2_spike/` and the oracle team's HLD V1–V5 are left untracked (not ours).
