## Open decisions for Arthur (non-blocking; build continues meanwhile)

- **DEC-3 (Stage 5 `thiserror` — SUPERVISOR DECIDED, reversible, low-stakes).** Devil's advocate flagged `thiserror = "2"` (proc-macro tree) pulled for ONE fieldless `ExtractError::NotImplemented` variant — the DEC-2/minimal-deps lesson. Although the parent brief lists `thiserror` as an allowed lib dep, the brief's governing PRINCIPLE is "minimal & conservative", and the crate is internally inconsistent (defers scraper/html5ever on minimal-deps grounds, pulls a derive-macro tree for one string). **Decision: drop `thiserror` for M1; hand-write `impl Display + std::error::Error` (~8 LOC).** Yields a genuinely **zero-dependency M1 library floor** (a real virtue for a crates.io lib's first cut). Trivially reversible: re-add `thiserror` at the milestone where `ExtractError` actually grows variants/`#[from]`/`#[source]` (noted in `lib.rs` doc), exactly as scraper/html5ever are deferred. Within autonomous scope (small dep/impl choice, principle-aligned, not architectural). Arthur: revert trivially if you prefer the listed dep.

- **DEC-2 (Stage 3 `fetch` HTTP mechanism — SUPERVISOR DECIDED, reversible, veto welcome).** `ureq` pulled 51 of 71 benchmark crates (full rustls + ICU tower + 2× webpki-roots) into the committed lockfile for a developer-only, out-of-band, run-~50-times GET that is provably never on the scoring path. Devil's advocate + code-reviewer both flagged; DA argued decisively it violates HLD §3 "smallest thing that works" and that the "everything in Rust" steer governs the harness's *own logic/report*, not throwaway scaffolding (HLD §6 explicitly calls fetch out-of-band). **Supervisor decision: replace `ureq` with `std::process::Command` shelling `curl` (ships Win10/11 + Git), README-documented.** Rationale: keeps the scoring path hermetic & dep-free; removes 50 crates of supply chain better undone now than carried through Stages 4–8; trades a pinned Rust dep for an unpinned system tool ONLY on the throwaway path; the committed snapshots (the real artifacts) are reproducible regardless of how curl behaved. Touches Arthur's "everything in Rust" steer → flagged. Trivially reversible (re-add `ureq`). **Arthur: veto → I restore `ureq`.**

- **DEC-1 (HLD §8 tokenizer spec — casefold).** Devil's advocate PROVED false-negatives: lowercase+NFC (the literal §8 spec) is NOT Unicode caseless matching — `tokens("ﬁle") != tokens("file")`, MICRO SIGN U+00B5 ≠ Greek mu, ﬀ/ﬃ ligatures split. These occur in our SEC/regulator corpus, so the "English/Western-European v1" scope only partially excuses it; Coverage/Precision get artefactually depressed (measuring-instrument bias). **Mitigation in place:** documented as a known limitation + pinning test `known_limitation_ligature_not_casefolded` (so a future fix is intentional/reviewed). **Decision needed:** change HLD §8 "lowercase, NFC" → **`NFKC_Casefold`** for v1?
