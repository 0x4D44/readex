### Iter 1 progress notes — root cause identified

After dumping both XMLs and aligning tokens:
- **Tokens 2390-2401** (Cardwell + Cole, with quotes): both Rust & Python emit
  `<item>^ <list rend="ul"><item>Cardwell<quote/></item><item>Cole<quote/></item></list></item>`. Identical.
- **Tokens 2402-2403**: Python emits a DUPLICATE
  `<item>Cardwell, Diane (...) Archived... December 30, 2016. </item>` — i.e.
  the text from the FIRST inner item of the previous list. Rust does NOT emit
  this duplicate.
- Token 2406-2415: env-reports list — both agree on the LOCAL shape, just
  shifted by 2 tokens because of the Cardwell duplicate above.

**Mechanism (empirically verified with lxml):** `iterdescendants("item")`
PRE-COMPUTES the next-pointer at yield time. The body of the iter recursively
calls `handle_lists` on a nested list, which sets inner items' `tag = "done"`.
But the precomputed next-pointer ALREADY points at the first inner item, so
the outer iter yields it AGAIN despite the tag mutation. Subsequent siblings
are skipped (tag check is current). So exactly ONE inner item gets a duplicate
emission per nested-list-in-item.

This is a Python/lxml quirk that the Rust port currently does NOT mimic.
Trafilatura faithfulness requires we replicate it.

**Empirical lxml proof:**
```
xml = '<list><item id="x"><list><item id="y1"/><item id="y2"/></list></item></list>'
for elem in tree.iterdescendants('item'):
    print(elem.get('id'), elem.tag)
    if elem.get('id') == 'x':
        for it in elem.iterdescendants('item'):
            it.tag = 'done'   # mutate y1, y2
# OUTPUT:
#   x item
#   y1 done    ← yielded despite mutation (precomputed)
# y2 NOT yielded (current tag check skips it)
```

**Fix shape:** after Rust's `handle_lists` processes an item that contained
a nested list, ALSO process the FIRST descendant item from that nested list
as a sibling at the outer level (mimicking the precompute). Skip subsequent
inner items (their tags are already "done", as today).

Implementing.

### Iter 1 — RESOLVED ✓ (Apple_Inc Wiki pinned, 10/10)

**Fix:** in `handle_lists`'s outer loop (Rust at `main_extractor.rs:266+`):
1. BEFORE the non-leaf else branch, snapshot the first descendant `<item>`'s
   text + has-children flag (`peek_first_descendant_item`).
2. Run process_nested_elements as normal.
3. After the gate/append/replace_element_tag steps, IF the snapshot has
   non-empty text AND had element-children, append a phantom `<item>`
   with text=snapshot to processed_element.

empirical lxml proof inlined in the code.
